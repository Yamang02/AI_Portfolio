# P02: css-module-migration

## 목표

- 전체 CSS Module 파일에서 `var(--color-*)`, `var(--spacing-*)`, `var(--border-radius-*)`, `var(--font-size-*)` 등 구형 변수 참조가 0개가 된다
- `bridge.css`가 삭제된다
- 주요 페이지(홈, 프로필, 프로젝트, 아티클, 챗봇)에서 라이트/다크 모드 시각 이상 없음이 확인된다

## 구현 상세

### 접근

1. 코드모드 스크립트(Node.js)로 CSS Module 내 구형 변수 참조를 일괄 교체
   - `--color-bg-primary` → `--ds-color-bg-base`
   - `--color-text-primary` → `--ds-color-text-primary`
   - `--spacing-{n}` → `--ds-spacing-{n}`
   - `--border-radius-{key}` → `--ds-radius-{key}`
   - `--shadow-{key}` → `--ds-shadow-{key}`
   - `--font-size-{key}` → `--ds-text-{key}` 등
2. 교체 결과 수동 검수 (예외 케이스 처리)
3. `bridge.css` import 제거 후 삭제
4. 페이지별 시각 검수

### 변경 파일

| 대상 | 작업 |
|------|------|
| `frontend/src/**/*.module.css` (약 80개) | 코드모드 일괄 교체 |
| `frontend/src/admin/**/*.css` | 동일 교체 |
| `frontend/src/index.css` | bridge.css import 제거 |
| `frontend/src/design-system/styles/bridge.css` | 삭제 |

### 교체 매핑표 (코드모드 입력)

| 구형 변수 | 신규 변수 |
|-----------|-----------|
| `--color-bg-primary` | `--ds-color-bg-base` |
| `--color-bg-secondary` | `--ds-color-bg-subtle` |
| `--color-bg-tertiary` | `--ds-color-bg-muted` |
| `--color-text-primary` | `--ds-color-text-primary` |
| `--color-text-secondary` | `--ds-color-text-secondary` |
| `--color-text-tertiary` | `--ds-color-text-muted` |
| `--color-border-default` | `--ds-color-border-default` |
| `--color-border-hover` | `--ds-color-border-hover` |
| `--color-primary` | `--ds-color-action-primary-default` |
| `--color-primary-hover` | `--ds-color-action-primary-hover` |
| `--color-primary-active` | `--ds-color-action-primary-active` |
| `--spacing-{n}` | `--ds-spacing-{n}` |
| `--border-radius-{key}` | `--ds-radius-{key}` |
| `--shadow-{key}` | `--ds-shadow-{key}` |
| `--font-size-{key}` | `--ds-text-{key}` |

## 체크리스트

- [ ] 코드모드 스크립트가 작성되고 dry-run 결과가 검토된다
- [ ] `frontend/src` 전체에서 `var(--color-` 패턴이 0건이다
- [ ] `frontend/src` 전체에서 `var(--spacing-` 패턴이 0건이다
- [ ] `frontend/src` 전체에서 `var(--border-radius-` 패턴이 0건이다
- [ ] `frontend/src` 전체에서 `var(--font-size-` 패턴이 0건이다
- [ ] `bridge.css` 파일이 존재하지 않는다
- [ ] 라이트 모드 홈/프로필/프로젝트/아티클/챗봇 페이지에 시각적 이상이 없다
- [ ] 다크 모드 홈/프로필/프로젝트/아티클/챗봇 페이지에 시각적 이상이 없다
- [ ] Admin 페이지(로그인, 테크스택 관리)가 라이트/다크 양쪽에서 정상 동작한다
