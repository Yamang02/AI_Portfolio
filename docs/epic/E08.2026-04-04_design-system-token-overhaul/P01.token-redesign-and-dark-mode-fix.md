# P01: token-redesign-and-dark-mode-fix

## 목표

- `globals.css`가 `--ds-*` / `--ui-*` 접두사 규칙을 따르는 새 CSS 변수 체계로 교체된다
- `bridge.css`가 추가되어 기존 `--color-*` 참조 컴포넌트가 P02 전까지 정상 동작한다
- MarkdownRenderer와 ChatBubble이 새 토큰을 소비하며 다크모드 가독성 문제가 해소된다
- TypeScript 토큰 파일이 새 CSS 변수명과 1:1 대응한다

## 구현 상세

### 접근

1. 신규 토큰 목록을 `globals.css` 형식으로 설계 (라이트 + 다크 양쪽)
2. 기존 `globals.css` 를 신규 토큰 체계로 전면 교체
3. `bridge.css` 작성 (`--color-bg-primary: var(--ds-color-bg-base)` 형태)
4. TS 토큰 파일 재정의 (CSS 변수명 일치)
5. 우선 수정 컴포넌트 2개 CSS Module 교체

### 신규 토큰 그룹

**Surface (배경 계층)**
```
--ds-color-bg-base
--ds-color-bg-subtle
--ds-color-bg-surface
--ds-color-bg-surface-high
--ds-color-bg-muted
```

**Markdown / 코드블록 전용**
```
--ds-color-bg-code
--ds-color-bg-code-block
--ds-color-border-code
```

**Chat 전용**
```
--ds-color-bg-chat-user
--ds-color-bg-chat-ai
--ds-color-text-chat-user
--ds-color-text-chat-ai
```

**Action 상태**
```
--ds-color-action-primary-default
--ds-color-action-primary-hover
--ds-color-action-primary-active
--ds-color-text-on-action-primary
```

**Shell 이동 (`--ui-*`)**
```
--ui-header-height
--ui-header-bg
--ui-footer-bg
--ui-hero-bg / --ui-hero-title / --ui-hero-subtext 등
--ui-featured-bg / --ui-featured-header 등
```

### 변경 파일

| 파일 | 작업 |
|------|------|
| `frontend/src/design-system/styles/globals.css` | 전면 재작성 |
| `frontend/src/design-system/styles/bridge.css` | 신규 생성 |

| `frontend/src/design-system/tokens/colors.ts` | CSS 변수명 일치 재정의 |

| `frontend/src/shared/ui/markdown/MarkdownRenderer.module.css` | 신규 토큰 적용, 다크모드 수정 |

## 체크리스트

- [ ] `globals.css` 내 `--color-*` 변수가 0개가 된다
- [ ] `globals.css` 내 `--breakpoint-*` 중복 선언이 제거된다
- [ ] `bridge.css`가 존재하며 기존 `--color-*` alias 전체를 포함한다
- [ ] `index.css`가 `bridge.css`를 import한다
- [ ] 라이트 모드에서 프로필 Markdown 렌더링이 시각적으로 이전과 동등하다
- [ ] 다크 모드에서 Markdown 코드블록 배경/테두리/텍스트 대비가 명확하다
- [ ] 다크 모드에서 ChatBubble 유저/AI 말풍선이 배경과 구분된다
- [ ] TypeScript 토큰 파일의 키가 대응 CSS 변수명과 1:1 매핑된다
