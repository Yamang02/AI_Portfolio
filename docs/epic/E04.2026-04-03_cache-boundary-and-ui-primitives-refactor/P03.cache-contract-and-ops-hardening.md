# P03: 캐시 운영·계약 검증 강화

## 목표

캐시 포맷/무효화/운영 절차가 현재 구현과 일치하고, 재배포 시 동일한 직렬화 회귀가 반복되지 않는다.

## 구현 상세

### 접근

- E02 문서의 직렬화·경계 설명을 현재 코드(`WRAPPER_ARRAY`, HTTP `@Primary` vs `redisObjectMapper`)와 일치시킨다.
- 운영 절차(플러시 시점, 검증 엔드포인트, 관측 로그)는 E02 Readme의 기존 운영 항목 및 P04 체크리스트와 연결한다.
- API 계약 스모크·캐시 검증 시나리오는 E02 P04 문서를 단일 참조로 둔다.
