# P02: content.json 실제 데이터 작성

## 목표

- `profile/public/content.json`이 P01에서 정의한 스키마를 따르며, `index.html`의 모든 실제 콘텐츠를 빠짐없이 담는다.
- placeholder 데이터가 남지 않는다.

## 구현 상세

### 소스

`profile/index.html`에 하드코딩된 콘텐츠를 기준으로 추출.

### 섹션별 추출 대상

#### hero
- `nameEn`: `"Lee JeongJun"`
- `nameKo`: `"이정준"`
- `roles`: `["Educator", "Developer", "Artist", "Entrepreneur"]`

#### identity
- `tagline`: `"스스로 구상하고 직접 만듭니다"`
- `cards`: 4개 역할 카드 (교육자 / 개발자 / 예술가 / 사업가) 각각의 icon, title, subTitle, body

#### activities (5개)
| id | modalLayout | 비고 |
|----|------------|------|
| `teaching` | `columns` | 상명고/배재고 2-column |
| `career` | `columns` | 교육/경력 2-column, itemStyle: 'career' |
| `artclub` | `site` | img 미디어, 외부 링크 있음 |
| `design` | `site` | video 미디어, 외부 링크 있음 |
| `vibe` | `none` | highlight 카드, body HTML 포함 |

#### interests (3개)
각각 icon(Material Symbol), title, body 추출.

### 파일 변경

| 파일 | 변경 |
|------|------|
| `profile/public/content.json` | 전체 재작성 (P01 스키마 적용, 실제 데이터) |
