# P04: cutover-validation-and-release-readiness

## 목표

- 데이터/백엔드/프런트 변경을 통합 검증해 컷오버 가능 상태를 만든다.
- 스테이징 기준에서 멀티사이트 노출 계약이 의도대로 동작하는지 확인한다.
- 운영 반영 전 회귀 리스크와 롤백 경로를 확정한다.

## 구현 상세

### 전제 조건

- P02, P03 구현이 완료되어야 한다.
- 스테이징 배포 기준과 도메인 매핑 기준(E16 Option B)을 재확인한다.

### 접근 방식

1. 데이터 검증
- `projects.readme` 이관 누락 여부 확인
- `project-overview` Article와 카드-아티클 연결 무결성 점검

2. 계약 검증
- 프로젝트 상세 API 응답 스냅샷/통합 테스트
- 프런트 상세 페이지 렌더링 회귀 테스트
- 사이트 컨텍스트별 노출 케이스 검증(`portfolio`, `business`)

3. 배포 준비
- 롤백 전략: 마이그레이션 실패/조회 오류/렌더링 오류별 복구 절차 고정
- 운영 체크리스트: 모니터링 포인트, 확인 URL, 즉시 대응 담당 정리

## 체크리스트

- [ ] 이관 데이터 샘플링 검증 완료
- [ ] API 통합 테스트 통과
- [ ] 프런트 회귀 테스트 통과
- [ ] 스테이징에서 사이트 컨텍스트별 노출 검증 완료
- [ ] 롤백 시나리오 문서화 완료
- [ ] 릴리즈 Go/No-Go 기준 합의 완료
