# P05: 스테이징 GCP DB 연결 (Cloud SQL Java Connector)

## 목표

- 스테이징 Cloud Run이 **GCP Cloud SQL for PostgreSQL**에 안정적으로 붙도록 백엔드 연결을 정리한다.
- **단일 JDBC URL 문자열(`POSTGRES_URL`)을 쓰지 않고**, 인스턴스·DB명·계정을 **환경 변수로 분리**한다.
- 공식 **Cloud SQL Java Connector**(`postgres-socket-factory`)로 JDBC를 조립한다.
- 헥사고날: 조립은 `infrastructure` (`DatabaseConfig`, `CloudSqlPostgresDataSourceFactory`), 도메인은 `DataSource`만 의존.

## 필수 환경 변수 (Cloud Run / GitHub Actions)

| 변수 | 예시 / 설명 |
|------|-------------|
| `GCP_CLOUD_SQL_INSTANCE_CONNECTION_NAME` | `yamang02-ai-portfolio:asia-northeast3:ai-portfolio-pg-staging` |
| `POSTGRES_DATABASE` | 애플리케이션 DB 이름 |
| `POSTGRES_USER` | DB 사용자 |
| `POSTGRES_PASSWORD` | GitHub **Secrets** 등으로 주입 |

워크플로: Repository **Variables**에 인스턴스명·DB명·유저, **Secrets**에 비밀번호.

## 범위 (In)

- `DatabaseConfig` + `CloudSqlDataSourceCondition`: **네 분리 변수가 모두 있을 때만** `DataSource` 등록 (프로필로 staging/production을 나누지 않음).
- `CloudSqlPostgresDataSourceFactory`: `cloudSqlInstance` + `socketFactory` + `cloudSqlRefreshStrategy=lazy`
- `pom.xml`: `com.google.cloud.sql:postgres-socket-factory`

**스테이징·프로덕션** 모두 동일하게 위 분리 변수 4개를 Cloud Run에 주입한다 (인스턴스·DB·계정 값만 환경별로 다름).

## 범위 (Out)

- 프로덕션 DB 컷오버 본작 (E17 P04)
- Hikari 풀 크기를 YAML로 세밀 튜닝 (후속: `@Bean`에서 `HikariConfig` 등)
