# P03: frontend-content-model-and-rendering

## 목표

- 프런트 타입 모델을 백엔드 계약(P02)과 동기화한다.
- 프로젝트 상세 페이지에서 `project-overview` 인라인 렌더링과 Technical Card 섹션을 안정적으로 표시한다.
- 멀티사이트 컨텍스트(`portfolio`, `business`)에 따른 노출 계약을 프런트 쿼리/필터에 반영한다.

## 구현 상세

### 전제 조건

- P02 API 계약이 고정되어야 한다.
- E15에서 확정한 FSD 경계와 페이지 구성 원칙을 유지한다.

### 접근 방식

1. 타입/쿼리 정렬
- `Project` 타입에서 `readme` 제거
- `projectOverviewArticle`, `technicalCards` 필드 추가
- Technical Card 카테고리/정렬 타입 고정

2. 프로젝트 상세 UI 갱신
- 개요(Overview) 아래 `project-overview` 인라인 렌더링 섹션 추가
- Technical Card 섹션 조건부 렌더링
- TOC(아코디언)와 카드 앵커 이동 연결

3. 콘텐츠 규칙 반영
- 카드에 연결된 상세 아티클 링크 동작 확정
- 사이트 컨텍스트 기준 노출 필터(`portfolio`, `business`)와 데이터 로더 정렬

### 변경 파일(예정)

- `frontend/src/main`의 프로젝트 상세 페이지/타입/쿼리 모듈
- 필요한 경우 공용 UI 컴포넌트(카드, TOC, 마크다운 렌더러 조합)

## 체크리스트

- [ ] `Project`, `Article`, `ProjectTechnicalCard` 타입 업데이트 완료
- [ ] 프로젝트 상세에서 `projectOverviewArticle` 인라인 렌더링 완료
- [ ] Technical Card 섹션/TOC 렌더링 및 앵커 이동 동작 확인
- [ ] 사이트 컨텍스트 기반 노출 조건 반영 완료
- [ ] 기존 프로젝트 상세 화면 회귀(콘텐츠 미존재 케이스 포함) 확인
