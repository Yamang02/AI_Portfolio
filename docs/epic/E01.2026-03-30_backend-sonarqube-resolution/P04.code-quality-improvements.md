## 구현 상세

### 접근 방식

- **S1192 (20건)**: 동일 리터럴 3회 이상 반복 시 `private static final` 상수 추출. 파일 내 또는 공통 상수 클래스에 배치
- **S3776 (16건)**: 루트 원인 분석 필수 — 단순 추출이 아닌 로직 단순화 우선. Early return, 가드 절, 전략 패턴 등 활용
- **S1168 (11건)**: `return null` → `return Collections.emptyList()` 또는 `List.of()`
- **S3358 (5건)**: 중첩 삼항 → if/else 변환
- **나머지**: 각 규칙의 권장 수정 패턴 적용

### 변경 범위

- S1192: 해당 파일에 상수 추출
- S3776: 복잡도 높은 메서드 분리 (가장 큰 변경 범위)
- S1168, S3358, S1141 등: 각 파일 내 국소적 수정
