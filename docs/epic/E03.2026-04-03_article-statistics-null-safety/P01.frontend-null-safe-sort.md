# P01: 프론트엔드 null-safe 정렬 수정

## 목표

null/undefined 필드를 가진 API 응답이 들어와도 정렬 연산에서 크래시가 발생하지 않는다.  
`ArticleListPage`와 `TechStackList` 두 컴포넌트가 ErrorBoundary 진입 없이 정상 렌더링된다.

## 구현 상세

### 접근

sort 비교 함수에서 필드에 직접 접근하는 패턴을 null-safe하게 수정.  
`null`/`undefined` 값은 빈 문자열(`''`) 또는 0으로 대체하여 정렬 순서를 보존.

동일 패턴으로 발견된 위험 지점:

| 위치 | 패턴 | 위험도 |
|------|------|--------|
| `ArticleListPage.tsx:118` | `a.title.localeCompare(b.title)` — `seriesTitle`이 null 가능 | **크래시 재현 중** |
| `TechStackList.tsx:110,117` | `a.name.toLowerCase()`, `a.name.localeCompare(b.name)` — `name`이 TechStack 객체로 전달 시 null 가능 | 잠재적 크래시 |

### 변경 범위

| 파일 | 변경 내용 |
|------|-----------|
| `frontend/src/main/pages/ArticleListPage.tsx` | `.sort((a, b) => a.title.localeCompare(b.title))` → `(a.title ?? '').localeCompare(b.title ?? '')` |
| `frontend/src/main/entities/tech-stack/ui/TechStackList.tsx` | sort 파라미터를 `({ name: nameA = '' }, { name: nameB = '' })` 로 destructuring — null-safety와 가독성을 동시에 확보, 본문에서 `a.name` 참조 제거 |

## 체크리스트

- [ ] `ArticleListPage` sort 비교 함수가 `(a.title ?? '').localeCompare(b.title ?? '')` 형태로 변경됨
- [ ] `TechStackList` sort 내 `name` 접근 전체가 `(a.name ?? '')` 형태로 방어됨
- [ ] `seriesTitle`이 null인 시리즈 항목이 포함된 응답에서 아티클 목록 페이지가 정상 렌더링됨
- [ ] `name`이 null인 TechStack 항목이 포함될 때 TechStackList가 정상 렌더링됨
- [ ] 정상 데이터 케이스에서 기존 정렬 동작 동일 (알파벳 오름차순, 핵심 기술 우선)
