# P02: 백엔드 seriesTitle null 원인 제거

## 목표

`GET /api/articles/statistics` 응답의 `series[].seriesTitle`이 null을 포함하지 않는다.  
`seriesId`는 존재하나 대응하는 `ArticleSeries` 엔티티가 없는 고아 참조가 통계에서 제외되거나, 엔티티가 있으나 title이 null인 경우 DB 정합성이 복구된다.

## 구현 상세

### 접근

1. **원인 파악**: `PostgresArticleRepository.getStatistics()`의 `seriesId` → `ArticleSeries` 조인 로직을 분석하여 null title 진입 경로 확정.
   - 고아 `seriesId` (Article에는 있지만 ArticleSeries 엔티티 없음) → 통계에서 필터링
   - `ArticleSeries.title`이 DB에서 null → DB 데이터 수정 또는 빈 문자열 fallback

2. **코드 수정**: null seriesTitle을 가진 `SeriesStatistics` 항목을 `getStatistics()` 구현에서 필터링하거나 방어적으로 처리.

3. **DB 점검** (운영 항목): staging DB에서 `seriesTitle`이 null인 Series 레코드 또는 고아 seriesId 보유 Article 확인.

### 변경 범위

| 파일 | 변경 내용 |
|------|-----------|
| `backend/src/main/java/com/aiportfolio/backend/infrastructure/persistence/postgres/adapter/PostgresArticleRepository.java` | `getStatistics()` 내 seriesStats 빌드 시 null title 항목 필터링 또는 방어 처리 |

## 체크리스트

- [ ] null `seriesTitle`을 유발하는 코드 경로가 확인됨 (고아 참조 vs DB null)
- [ ] `getStatistics()` 구현이 null `seriesTitle`을 가진 항목을 통계 응답에 포함하지 않음
- [ ] 정상 seriesTitle을 가진 항목은 응답에 그대로 포함됨
- [ ] staging DB에서 null seriesTitle 또는 고아 seriesId 현황 파악 및 기록
