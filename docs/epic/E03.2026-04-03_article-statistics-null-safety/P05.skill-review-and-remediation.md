# P05: 스킬 기반 검토 및 반영

## 전제 조건

**P01~P04 구현이 끝난 뒤** 착수한다. 코드 변경은 "검토에서 발견된 항목"에 한정한다.

## 목표

E03 변경 범위(프론트엔드 sort 수정, 백엔드 null-guard)에 basis 검토 스킬을 적용하고 결과를 기록한다.  
치명도가 낮은 개선은 이 Phase에서 코드·에픽 문서에 반영하고, 범위 밖 대형 리팩터는 백로그로 넘긴다.

## 적용 스킬 (역할)

| 스킬 `name` | 역할 |
|-------------|------|
| `cc-01-logic-level` | sort 비교 함수, null-guard 패턴 등 **순수 로직·계산 흐름** |
| `cc-05-code-hygiene` | 주석 위치·미사용 코드·import 위반 여부 |
| `file-basis-review` | **파일 단위** CC-01·CC-05 종합 패스 (변경 파일 전체) |

## 구현 상세

### 검토 범위 (파일)

| 파일 | 분류 |
|------|------|
| `frontend/src/main/pages/ArticleListPage.tsx` | P01 변경 — 필수 |
| `frontend/src/main/entities/tech-stack/ui/TechStackList.tsx` | P01 변경 — 필수 |
| `backend/.../PostgresArticleRepository.java` (`getStatistics` 메서드 주변) | P02 변경 — 필수 |

### 스킬별 산출물

| 스킬 | 기록할 내용 |
|------|-------------|
| `cc-01-logic-level` | null-guard 패턴이 CC-01 흐름 규칙을 따르는지 (early return·변환 방향·단순성) |
| `cc-05-code-hygiene` | 위반 0건 또는 수정 내역 (불필요 주석·미사용 import) |
| `file-basis-review` | 대상 파일별 Pass / 수정 완료 |

### 반영 규칙

- **코드:** CC-05-03/04 (dead store·unused import), 명백한 네이밍 정도만 수정.
- **에픽 문서:** 에픽 `Readme.md`의 목표·특이점이 구현 결과와 어긋나면 한 줄 보강.
- 동작 변경이 큰 리팩터는 하지 않는다.
