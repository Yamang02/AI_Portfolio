# P05: basis·스킬 검토 및 반영

## 전제 조건

**P01~P04 구현이 끝난 뒤** 착수한다. 코드 변경은 "검토에서 발견된 항목"에 한정한다.

## 목표

E03 변경 범위(프론트엔드 sort 수정, 백엔드 null-guard)에 basis 문서와 Cursor 스킬을 적용하고 결과를 기록한다.  
치명도가 낮은 개선은 이 Phase에서 코드·에픽 문서에 반영하고, 범위 밖 대형 리팩터는 백로그로 넘긴다.

## 적용 자료 (역할)

| 구분 | 경로 / 식별자 | 역할 |
|------|----------------|------|
| Basis 문서 | [`docs/basis/coding/CC-01-logic-level-rules.md`](../../basis/coding/CC-01-logic-level-rules.md) | sort 비교·null-guard 등 **순수 로직·데이터 흐름** (CC-01) |
| Cursor 스킬 | `.cursor/skills/cc-05-code-hygiene` (`cc-05-code-hygiene`) | 주석 위치·미사용 코드·import |
| Cursor 스킬 | `.cursor/skills/file-basis-review` (`file-basis-review`) | 변경 파일 단위 CC-01·CC-05 종합 패스 |

> CC-01은 저장소의 **basis 마크다운 문서**로 관리된다. 스킬 이름 `cc-01-logic-level`과 동일 규칙 집합이 아닐 수 있으므로, 검토 시 위 Basis 문서를 기준으로 한다.

## 검토 결과

### CC-01 (basis)

- **ArticleListPage / TechStackList:** 정렬은 `compareStrings` 단일 경로로 위임되어 분기가 단순하다. null/undefined는 유틸 내부에서 `''`로 수렴한다.
- **PostgresArticleRepository:** `series.getTitle()` null 시 `""`로 치환하여 JSON null 전파를 차단; 고아 `seriesId`는 기존과 같이 `series == null`에서 제외된다.

### CC-05 (스킬)

- 변경 파일에서 불필요 import·dead code 추가 없음.
