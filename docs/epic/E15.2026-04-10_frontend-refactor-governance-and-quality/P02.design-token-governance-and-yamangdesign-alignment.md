# P02: design-token-governance-and-yamangdesign-alignment

## 목표

- `YamangDesign` 기준의 토큰 아키텍처를 `AI_PortFolio`에 맞게 재정의한다.
- `frontend`와 `profile`이 같은 naming/governance 규칙을 사용하도록 정리한다.
- legacy token 이름, duplicated theme constant, inline hex usage를 migration 가능한 형태로 축소한다.
- token 결정 중 재사용 가능한 부분은 이후 `policy_compiler`로 이식 가능한 상태로 만든다.

## 구현 상세

### 1. AI_PortFolio token layer 결정

- Primitive / Global scale
- Semantic / Alias token
- UI shell token

기본 적용 범위는 위 3단계로 제한한다. reusable external component library를 별도 publish하지 않는 한, component token tier를 먼저 늘리지 않는다.

### 2. Naming 규칙

- prefix는 `--ds-*`, `--ui-*`를 사용한다.
- role-based naming을 사용한다.
- taxonomy는 `Namespace -> Object -> Base -> Modifier` 순서를 따른다.
- UI 코드에서는 `var(--ds-*)`, `var(--ui-*)`만 사용한다.

### 3. 정리 대상

| 대상 | 현재 문제 | P02 결과 |
|------|-----------|-----------|
| `frontend/src/design-system/styles/globals.css` | `--color-*`, `--breakpoint-*` 중심 | canonical token variables로 정리 |
| `frontend/src/design-system/tokens/colors.ts` | legacy variable reference와 1:1 mapping 불일치 | CSS variable contract와 동기화 |
| `frontend/src/shared/config/theme.ts` | token-like constant 중복 | token consumer 또는 thin mapping으로 축소 |
| `profile/index.html` | inline Tailwind theme + hex literal 다수 | build-time token consumption으로 교체 |
| `frontend/src/**/*.module.css` | 파일별 direct hex/legacy token 잔존 | semantic/ui token 사용으로 통일 |

### 4. Migration 방식

1. canonical token dictionary를 먼저 정의한다.
2. legacy token alias를 bridge layer로 한시 운영한다.
3. `frontend` CSS Modules와 shared UI를 semantic token으로 이관한다.
4. `profile`의 inline theme source를 제거하고 typed/build-time token source로 전환한다.
5. bridge 제거 시점을 문서화하고 실제 참조 0을 확인한 뒤 삭제한다.

### 5. 허용 / 금지

- 허용: token definition, palette mapping, preset 생성 파일 안의 direct hex/rgb
- 금지: page/component/css module/tsx 내부 direct hex/rgb
- 허용: migration 중 deprecation alias
- 금지: 영구적인 legacy token 공존

## 체크리스트

- [ ] `AI_PortFolio`의 token layer 범위가 Primitive + Semantic + UI shell로 확정되었다.
- [ ] `--ds-*`, `--ui-*` naming 규칙이 문서화되었다.
- [ ] UI 코드에서 hex/rgb literal 사용 금지 범위가 정의되었다.
- [ ] `frontend/src/design-system/styles/globals.css`가 canonical token source 역할로 정리된다.
- [ ] `frontend/src/shared/config/theme.ts`의 duplicated primitive/theme constant가 축소된다.
- [ ] `profile/index.html`의 inline Tailwind theme source가 제거 대상로 명시되었다.
- [ ] bridge alias의 도입 조건과 제거 조건이 문서화되었다.
