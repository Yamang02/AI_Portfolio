# P01: shared-infrastructure-migration

## 목표

- `admin/api/adminApiClient.ts`, `admin/api/adminUploadApi.ts`, `admin/api/types.ts`, `admin/types/dataTable.types.ts`, `admin/utils/dataTransformers.ts`를 `admin/shared/` 안으로 이동한다.
- 이동 후 모든 import 경로가 갱신되고 `npm --prefix frontend run build`가 통과한다.
- `admin/api/`(auth/project api 제외), `admin/types/`, `admin/utils/` 폴더가 삭제된다.

## 구현 상세

### 이동 매핑

| 현재 경로 | 이동 후 경로 |
|---|---|
| `admin/api/adminApiClient.ts` | `admin/shared/api/adminApiClient.ts` |
| `admin/api/adminUploadApi.ts` | `admin/shared/api/adminUploadApi.ts` |
| `admin/api/types.ts` (ApiError) | `admin/shared/types/apiError.ts` |
| `admin/types/dataTable.types.ts` | `admin/shared/types/dataTable.types.ts` |
| `admin/utils/dataTransformers.ts` | `admin/shared/lib/dataTransformers.ts` |

### 접근 방식

1. `admin/shared/api/`, `admin/shared/types/`, `admin/shared/lib/` 디렉토리 생성 (이미 있으면 재사용).
2. 각 파일을 새 경로로 이동 후 내부 상대 import를 수정.
3. 이동한 파일들을 import하는 모든 파일의 경로를 grep으로 탐색해 일괄 업데이트.
4. `admin/shared/api/index.ts`, `admin/shared/types/index.ts`, `admin/shared/lib/index.ts` barrel에 export 추가.
5. `admin/api/`에 auth/project api만 남은 상태 확인 (P02에서 처리).
6. `admin/types/`, `admin/utils/` 폴더 삭제.
7. 업로드 API 중복(`shared/api/uploadApi.ts` vs `adminUploadApi.ts`)을 확인해 단일 API로 수렴한다.
   - 원칙: `admin/shared/api`에서 업로드 API 엔트리를 하나만 유지한다.
   - `hooks/useUpload.ts`, `shared/ui/markdown/MarkdownEditor.tsx`가 동일한 업로드 계약을 사용하도록 맞춘다.

### 변경 범위 (파악 필요 파일)

- `adminApiClient.ts`를 import하는 모든 `admin/api/*Api.ts` 파일
- `dataTable.types.ts`를 import하는 `admin/hooks/useTablePage.ts`, `admin/shared/hooks/*`
- `dataTransformers.ts`를 import하는 project-management feature UI
- 업로드 API를 참조하는 `admin/hooks/useUpload.ts`, `admin/shared/ui/markdown/MarkdownEditor.tsx`
