# P05: admin-index-centric-import-alignment

## 목표

- `main` 레이어와 동일하게 `admin`도 index(barrel) 중심 import 구조로 정리한다.
- 상대 경로 깊은 import와 내부 파일 직접 참조를 줄여 레이어별 public API를 명확히 한다.
- 정리 후 `npm --prefix frontend run build`와 `npm --prefix frontend run test:coverage`가 통과한다.

## 구현 상세

### 정렬 원칙

1. 레이어별 공개 진입점(`index.ts`)을 우선 사용한다.
2. 다른 레이어에서 내부 구현 파일을 직접 import하지 않는다.
3. 같은 레이어 내부에서는 상대 경로 import를 허용하되, 상위 레이어로의 역참조는 금지한다.
4. 기존 절대 경로 alias(`@/admin/...`) 사용 시에도 가능한 경우 `@/admin/<layer>`의 index export를 통해 참조한다.

### 적용 우선순위

1. `features/auth`와 `entities/project`처럼 참조가 많은 모듈부터 index export를 확정한다.
2. `shared/hooks`, `shared/api`, `shared/lib`, `shared/types`의 공개 API를 정리한다.
3. `pages/*`, `app/*`, `features/*`에서 내부 파일 직접 참조를 index 경유 import로 교체한다.
4. 사용되지 않는 중간 re-export나 중복 alias 경로를 제거한다.

### 변경 범위

- `admin/entities/**/index.ts`
- `admin/features/**/index.ts`
- `admin/shared/**/index.ts`
- `admin/pages/**`, `admin/app/**` import 구문
