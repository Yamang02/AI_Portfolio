# P03: hooks-and-auth-feature-migration

## 목표

- `admin/hooks/` 내 나머지 파일(`useAuth.ts`, `useAdminQuery.ts`, `useAdminMutation.ts`, `useTablePage.ts`, `useUpload.ts`)을 FSD 레이어 안으로 이동한다.
- `useTablePage.ts`와 `shared/hooks/useManagementPage.ts`의 역할 중복을 확인하고 단일화한다.
- `admin/hooks/` 폴더가 완전히 삭제된다.

## 구현 상세

### 이동 매핑

| 현재 경로 | 이동 후 경로 | 비고 |
|---|---|---|
| `hooks/useAuth.ts` | `features/auth/model/useAuth.ts` | AuthProvider는 `features/auth/ui/`로 분리 검토 |
| `hooks/useAdminQuery.ts` | `shared/hooks/useAdminQuery.ts` | |
| `hooks/useAdminMutation.ts` | `shared/hooks/useAdminMutation.ts` | |
| `hooks/useTablePage.ts` | `shared/hooks/useTablePage.ts` 또는 삭제 | 현재 사용처 없음을 확인 후 결정 |
| `hooks/useUpload.ts` | `shared/hooks/useUpload.ts` | |

### useAuth 분리 방안

`useAuth.ts`에는 `AuthProvider`(React 컴포넌트)와 `useAuth`(hook)이 한 파일에 혼재한다.

- `AuthProvider` → `features/auth/ui/AuthProvider.tsx`
- `useAuth` + `AuthContext` → `features/auth/model/useAuth.ts`

단, 두 파일의 분리가 순환 참조를 만들지 않는지 확인 후 결정. 문제가 있으면 한 파일 유지.

### useTablePage vs useManagementPage

두 hook의 시그니처와 동작을 비교한다:
- 동일한 역할이면 `useManagementPage`를 유지하고 `useTablePage` 참조를 교체 후 삭제.
- 역할이 다르면 둘 다 `shared/hooks/`에 두고 이름으로 명확히 구분.
- 현재 `useTablePage.ts` 실제 사용처가 없으면(0 reference) `shared`로 이동하지 않고 삭제를 우선한다.

### 변경 범위

- `useAuth`를 import하는 `features/auth/ui/ProtectedRoute.tsx`, `features/auth/ui/LoginForm.tsx`, `app/AdminRootGate.tsx`, `shared/components/AdminLayout.tsx`
- `AuthProvider`를 import하는 `admin.tsx`
- `useAdminQuery`, `useAdminMutation`을 import하는 entity api 파일들
- `useTablePage`를 import하는 파일(없다면 삭제 검토)
- `useUpload`를 import하는 project-management feature UI
- `shared/hooks/index.ts` barrel에 신규 hook export 추가
