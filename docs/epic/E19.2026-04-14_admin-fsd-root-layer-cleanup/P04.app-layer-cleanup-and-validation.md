# P04: app-layer-cleanup-and-validation

## 목표

- `admin/config/queryClient.ts`를 `admin/app/config/queryClient.ts`로 이동한다.
- `admin/App.css`를 `admin/app/styles/App.css`로 이동하거나, 미사용이 확정되면 삭제한다.
- `admin/config/` 폴더가 삭제된다.
- `admin/` 루트에 FSD 6레이어(`app/`, `pages/`, `widgets/`, `features/`, `entities/`, `shared/`)만 남는다.
- 전체 빌드와 테스트가 통과하고 E19 목표 달성을 검증한다.

## 구현 상세

### 이동 매핑

| 현재 경로 | 이동 후 경로 |
|---|---|
| `admin/config/queryClient.ts` | `admin/app/config/queryClient.ts` |
| `admin/App.css` | `admin/app/styles/App.css` (사용 시) |

### config 이동

- `adminQueryClient`를 import하는 실제 엔트리 파일(`frontend/src/admin.tsx`)의 경로를 업데이트.
- `admin/config/` 폴더 삭제.

### App.css 이동

1. `App.css` 사용처를 먼저 확인한다.
2. 사용처가 있으면 `admin/app/styles/App.css`로 이동 후 import 경로를 업데이트한다.
3. 사용처가 없으면 `App.css`를 삭제하고 "미사용 스타일 제거"로 기록한다.

### 최종 루트 레이아웃 검증

E19 완료 조건:

```
admin/
├── app/        ✅
├── pages/      ✅
├── widgets/    ✅ (비어있어도 레이어로 유지)
├── features/   ✅
├── entities/   ✅
└── shared/     ✅
```

위 6개 외 다른 폴더/파일이 없어야 한다.

### 회귀 검증

- 전체 빌드: `npm --prefix frontend run build`
- 전체 테스트: `npm --prefix frontend run test:coverage`
- Admin 주요 화면(로그인, 프로젝트 목록, 기술스택, 프로필) 수동 동작 확인
