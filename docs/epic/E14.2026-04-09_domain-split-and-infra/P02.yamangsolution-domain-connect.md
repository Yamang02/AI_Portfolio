# P02: yamangsolution.com 도메인 연결

## 목표

- `www.yamangsolution.com`이 기존 frontend production CloudFront(`E384L5ALEPZ14U`)를 서빙한다
- `staging.yamangsolution.com`이 기존 frontend staging CloudFront(`E7KKBCETIHDH6`)를 서빙한다
- 두 도메인 모두 HTTPS가 정상 동작한다
- 기존 frontend 배포 워크플로우 환경변수가 `yamangsolution.com` 기준으로 업데이트된다

## 구현 상세

### 접근 (AWS CLI + Cloudflare 순서)

1. **ACM 인증서 발급** (us-east-1, DNS 검증)
   - 도메인: `yamangsolution.com`, `www.yamangsolution.com`, `staging.yamangsolution.com`
   - 또는 `yamangsolution.com` + `*.yamangsolution.com` 와일드카드로 한 번에 발급

2. **DNS 검증 처리**
   - AWS CLI로 검증 CNAME 값 조회
   - Cloudflare DNS에 CNAME 레코드 추가
   - 인증서 `Issued` 상태 대기

3. **기존 production CloudFront(`E384L5ALEPZ14U`) 업데이트**
   - Alternate domains 추가: `www.yamangsolution.com`, `yamangsolution.com`
   - 새 ACM 인증서 연결 (기존 `yamang02.com` 인증서는 `admin.yamang02.com` 때문에 병행 유지 불가 → ViewerCertificate 교체 방식 확인 필요)

4. **기존 staging CloudFront(`E7KKBCETIHDH6`) 업데이트**
   - Alternate domains 추가: `staging.yamangsolution.com`
   - 새 ACM 인증서 연결

5. **Cloudflare DNS 설정**
   - `www.yamangsolution.com` CNAME → `d37el02r3z3c70.cloudfront.net`
   - `yamangsolution.com` CNAME (또는 루트) → 동일
   - `staging.yamangsolution.com` CNAME → `duyanz9ufv3r7.cloudfront.net`
   - Proxy: **DNS only (회색 구름)** — CloudFront 자체 CDN 활용
   - SSL/TLS 모드: Full

6. **GitHub production/staging environment variables 업데이트**
   - `VITE_SITE_URL` (production) → `https://www.yamangsolution.com`
   - `VITE_SITE_URL` (staging) → `https://staging.yamangsolution.com`

7. **frontend 재배포 실행** (sitemap 등 URL 반영)

### 특이점

- CloudFront 하나에 ACM 인증서는 하나만 연결 가능 → `yamangsolution.com` 인증서가 `admin.yamang02.com`도 커버해야 함 (SAN에 포함 또는 별도 `admin.yamang02.com` CF 분리 검토)
- Cloudflare 루트 도메인(`yamangsolution.com`) CNAME은 CNAME Flattening으로 처리됨 (Cloudflare 자동 지원)

### 변경 범위

| 파일 | 작업 |
|------|------|
| `.github/workflows/frontend-production-aws.yml` | `VITE_SITE_URL` 변수 확인 |
| `.github/workflows/frontend-staging-aws.yml` | `VITE_SITE_URL` 변수 확인 |
| AWS (CLI) | ACM 인증서, CloudFront ×2 alternate domain 수정 |
| Cloudflare | DNS CNAME 3개 추가 |
| GitHub Settings | `VITE_SITE_URL` production/staging |
