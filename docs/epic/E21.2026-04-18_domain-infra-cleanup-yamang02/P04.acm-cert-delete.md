# P04: acm-cert-delete

## 목표

`yamangsolution.com` 관련 ACM 인증서가 AWS에 하나도 남아있지 않고,
불필요한 `yamang02.com` 중복 인증서도 정리된다.

## 구현 상세

### 삭제 대상 (P02·P03 완료 후 InUseBy가 비어있음을 확인하고 삭제)

| ARN (suffix) | 도메인 | 현재 상태 |
|---|---|---|
| `36da10e5` | `yamangsolution.com` + `*.yamangsolution.com` | 미사용 |
| `c71ef080` | `yamangsolution.com` + `admin.yamang02.com` + `*.yamangsolution.com` | P02 완료 후 미사용 |
| `168a4ce8` | `staging.yamangsolution.com` + `admin.staging.yamang02.com` | 미사용 |
| `1ef36112` | `staging.yamangsolution.com` + `staging.admin.yamang02.com` | P03 완료 후 미사용 |
| `1a130a89` | `staging.yamang02.com` | 미사용 (P03에서 활용 안 할 경우) |
| `7389ab9f` | `yamang02.com` (중복) | 미사용 |

### 삭제 방법

각 인증서 `InUseBy`가 빈 배열임을 CLI로 확인 후 삭제:
```
aws acm delete-certificate --region us-east-1 --certificate-arn <ARN>
```
