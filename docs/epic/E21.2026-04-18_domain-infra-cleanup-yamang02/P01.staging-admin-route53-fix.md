# P01: staging-admin-route53-fix

## 목표

`staging.admin.yamang02.com`이 CloudFront E7KKBCETIHDH6에 정상 연결되어 staging admin에 HTTP 200으로 접근된다.

## 구현 상세

### 문제

Route53에 `admin.staging.yamang02.com` A record(alias → duyanz9ufv3r7.cloudfront.net)가 존재하나,
CF E7KKBCETIHDH6의 실제 alias는 `staging.admin.yamang02.com`(`{환경}.{모듈}` 패턴).
CF는 Host 헤더가 alias 목록과 일치하지 않으면 403을 반환하므로, 현재 `admin.staging.yamang02.com`으로 접근하면 불통.

### 변경 범위

- Route53 hosted zone `Z01437241DGHDFQHEDV82` (`yamang02.com`)
  - 삭제: `admin.staging.yamang02.com` A alias record
  - 추가: `staging.admin.yamang02.com` A alias record → `duyanz9ufv3r7.cloudfront.net`

### 적용 방법

AWS CLI로 직접 적용 (Terraform import는 P05에서 처리).

## 체크리스트

- [x] `admin.staging.yamang02.com` Route53 A record 삭제 확인
- [x] `staging.admin.yamang02.com` Route53 A record 생성 확인 (alias target: duyanz9ufv3r7.cloudfront.net)
- [ ] `https://staging.admin.yamang02.com` 접속 시 200 응답 확인 — CF alias 불일치 해소 후 P03 완료 시 최종 확인
