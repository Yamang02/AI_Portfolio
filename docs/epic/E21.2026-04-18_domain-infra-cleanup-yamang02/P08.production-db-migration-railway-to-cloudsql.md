# P08: production-db-migration-local-docker-to-cloudsql

## 목표

로컬 Docker PostgreSQL의 데이터가 Cloud SQL production으로 이전되고,
Cloud Run production 서비스가 Cloud SQL을 바라보며 정상 운영된다.

## 구현 상세

### 마이그레이션 순서

1. 로컬 Docker PostgreSQL 컨테이너에서 pg_dump 수행
2. Cloud SQL Auth Proxy를 통해 Cloud SQL에 import
3. Cloud Run 환경변수 교체 (`POSTGRE_URL` → Cloud SQL 방식)
4. Cloud Run cloudsql-instances annotation 추가
5. production 재배포 후 동작 확인
6. Railway 연결 정보 환경변수 제거

### 1. 로컬 Docker에서 dump

```bash
# 로컬 컨테이너명 확인 후 실행
docker exec <postgres-container> pg_dump -U <user> -d <dbname> \
  --no-owner --no-acl -F c -f /tmp/prod_dump.dump

docker cp <postgres-container>:/tmp/prod_dump.dump ./prod_dump.dump
```

### 2. Cloud SQL Auth Proxy 경유 import

```bash
# Cloud SQL Auth Proxy 실행 (별도 터미널)
cloud-sql-proxy yamang02-ai-portfolio:asia-northeast3:ai-portfolio-pg-production \
  --port 5433

# 덤프 복원
pg_restore -h 127.0.0.1 -p 5433 -U <user> -d <dbname> \
  --no-owner --no-acl -F c ./prod_dump.dump
```

### 3. Cloud Run 환경변수 변경

**제거:**
```
POSTGRE_URL
```

**추가 (staging 패턴 동일):**
```
GCP_CLOUD_SQL_INSTANCE_CONNECTION_NAME=yamang02-ai-portfolio:asia-northeast3:ai-portfolio-pg-production
POSTGRES_DATABASE=<db명>
POSTGRES_USER=<앱유저>
POSTGRES_PASSWORD=<패스워드>
FLYWAY_USER=<마이그레이션 유저>
FLYWAY_PASSWORD=<패스워드>
```

**Cloud Run annotation 추가:**
```
run.googleapis.com/cloudsql-instances: yamang02-ai-portfolio:asia-northeast3:ai-portfolio-pg-production
```

### application-production.yml 확인 사항

staging과 동일하게 `GCP_CLOUD_SQL_INSTANCE_CONNECTION_NAME` 환경변수를 읽는지 확인.
`POSTGRE_URL` 방식을 별도로 처리하는 코드가 있다면 제거.

## 로컬에서 실행할 명령어 순서

```bash
# 1. 로컬 Docker에서 덤프
docker exec <postgres-container> pg_dump -U <user> -d <dbname> \
  --no-owner --no-acl -F c -f /tmp/prod_dump.dump
docker cp <postgres-container>:/tmp/prod_dump.dump ./prod_dump.dump

# 2. Cloud SQL에 복원 (public IP 임시 허용 상태)
pg_restore -h 34.47.78.133 -p 5432 \
  -U postgres -d postgres_production \
  --no-owner --no-acl -F c ./prod_dump.dump

# 3. 앱 유저 권한 부여 (psql로 접속)
psql -h 34.47.78.133 -U postgres -d postgres_production -c \
  "GRANT ALL ON ALL TABLES IN SCHEMA public TO portfolio_app_production;
   GRANT ALL ON ALL SEQUENCES IN SCHEMA public TO portfolio_app_production;
   GRANT ALL ON SCHEMA public TO portfolio_app_production;"

# 4. 데이터 검증 후 authorized-networks 제거 (public IP 접근 차단)
gcloud sql instances patch ai-portfolio-pg-production \
  --project yamang02-ai-portfolio \
  --clear-authorized-networks

# 5. Cloud Run 환경변수 교체 (데이터 복원 완료 후)
gcloud run services update ai-portfolio \
  --project yamang02-ai-portfolio --region asia-northeast3 \
  --add-cloudsql-instances yamang02-ai-portfolio:asia-northeast3:ai-portfolio-pg-production \
  --update-env-vars "GCP_CLOUD_SQL_INSTANCE_CONNECTION_NAME=yamang02-ai-portfolio:asia-northeast3:ai-portfolio-pg-production,POSTGRES_DATABASE=postgres_production,POSTGRES_USER=portfolio_app_production,POSTGRES_PASSWORD=iJAxGo5TInGVQ0lRky9xR6kNT2Z4ewZg" \
  --remove-env-vars "POSTGRE_URL"
```
