# P06: cloudflare-dns-cleanup

## 목표

Cloudflare `yamangsolution.com` DNS에서 AI Portfolio CloudFront를 가리키던 구 레코드가 제거되고,
`yamangsolution.com`이 YamangSolution 비즈니스 페이지만을 서빙한다.

## 구현 상세

### 배경

`yamangsolution.com`은 Cloudflare에서 권한 DNS를 전담하고 있다(Route53에 hosted zone 없음).
현재 Cloudflare에 CloudFront를 가리키는 레코드가 남아있을 수 있으며, 이를 정리한다.

### 확인 및 삭제 대상 (Cloudflare 콘솔에서 직접 확인)

Cloudflare Dashboard → `yamangsolution.com` → DNS 탭에서 아래 레코드 존재 여부 확인:

| 이름 | 타입 | 예상 값 |
|---|---|---|
| `yamangsolution.com` (apex) | A 또는 CNAME | CloudFront domain |
| `www` | CNAME | CloudFront domain |
| `staging` | CNAME | CloudFront domain |
| `admin` | CNAME | CloudFront domain |

위 레코드 중 CloudFront(`.cloudfront.net`)를 가리키는 것은 **모두 삭제**.

### YamangSolution 비즈니스 페이지 연결 (배포 완료 후)

YamangSolution(`c:\Users\ljj02\Desktop\dev\YamangSolution`) 배포 인프라가 확정되면:
- `www.yamangsolution.com` → YamangSolution 배포 엔드포인트로 CNAME 추가
- `yamangsolution.com` (apex) → Cloudflare CNAME Flattening 활용 또는 A record

### 주의

- Cloudflare Proxy(오렌지 클라우드)를 사용할 경우 YamangSolution 인프라가 Cloudflare SSL을 처리하므로 별도 ACM 불필요.
- CloudFront + Cloudflare Proxy 이중 프록시는 하지 않는다(SSL 충돌).

## 체크리스트

- [ ] Cloudflare `yamangsolution.com` DNS 탭에서 CloudFront 가리키는 레코드 전수 확인
- [ ] 해당 레코드 모두 삭제 확인
- [ ] `yamangsolution.com` / `www.yamangsolution.com` 접속 시 AI Portfolio CF로 라우팅되지 않음 확인
- [ ] YamangSolution 배포 후 `www.yamangsolution.com` 신규 레코드 추가 확인 (배포 완료 시점에 처리)
