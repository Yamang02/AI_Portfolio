# P03: staging-cf-alias-and-cert-cleanup

## 목표

CloudFront E7KKBCETIHDH6(staging admin)의 alias가 `staging.admin.yamang02.com` 단독으로 남고,
`staging.yamangsolution.com` alias가 제거되며, 인증서가 `*.yamang02.com`(f3d39247)으로 교체된다.

## 구현 상세

### 현재 상태

- CF E7KKBCETIHDH6 aliases: `staging.yamangsolution.com`, `staging.admin.yamang02.com`
- 사용 중인 ACM cert: `1ef36112` (`staging.yamangsolution.com` + `staging.admin.yamang02.com`)

### 변경 범위

- CF E7KKBCETIHDH6
  - aliases: `staging.yamangsolution.com` 제거 → `staging.admin.yamang02.com` 단독
  - ACM cert: `1ef36112` → `f3d39247` (`*.yamang02.com`)

### 주의

`*.yamang02.com` wildcard는 1단계 서브도메인(`admin.yamang02.com`)만 커버하고
2단계 서브도메인(`staging.admin.yamang02.com`)은 커버하지 않는다.
그러나 `f3d39247` cert의 SANs에 `*.yamang02.com`만 있으므로 staging admin에는 별도 SAN 추가가 필요.

→ 옵션 A: `staging.admin.yamang02.com` SAN을 `f3d39247` cert에 추가 후 교체
→ 옵션 B: `staging.yamang02.com` cert(1a130a89, 미사용)를 확인하고 SAN 추가하여 별도 사용
→ 권장: **옵션 A** — wildcard cert 단일화로 관리 포인트 최소화

### 적용 방법

1. ACM `f3d39247`에 `staging.admin.yamang02.com` SAN 추가 (새 cert로 재발급)
2. Route53에 새 cert DNS 검증 CNAME 추가
3. 검증 완료 후 CF E7KKBCETIHDH6 업데이트 (alias 제거 + 인증서 교체 동시)
