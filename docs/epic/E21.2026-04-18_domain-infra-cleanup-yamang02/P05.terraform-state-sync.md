# P05: terraform-state-sync

## 목표

Terraform production·staging 코드가 실제 AWS 상태를 완전히 반영하며,
`terraform plan`이 no-change를 반환한다.

## 구현 상세

### 현재 괴리

| 항목 | Terraform 코드 | 실제 상태 |
|---|---|---|
| production domain_name | `yamangsolution.com` | `yamang02.com` |
| production CF aliases | yamangsolution.com 계열 3개 | `admin.yamang02.com` 단독 |
| production ACM cert | `aws-dns` 모듈 생성 cert | `f3d39247` (`*.yamang02.com`) |
| staging CF aliases | `staging.yamangsolution.com` + `staging.admin.yamang02.com` | `staging.admin.yamang02.com` 단독 |
| profile prod CF (E22O2QL7DWQJDY) | 미존재 | `www.yamang02.com`, `yamang02.com` 서빙 |
| profile staging CF (E2KRLTT5DRFTUA) | 미존재 | `staging.yamang02.com` 서빙 |
| `aws-dns` 모듈 Route53 zone | `yamangsolution.com` zone 생성 | `yamang02.com` zone만 존재 |

### 변경 범위

**environments/production/main.tf**
- `module "dns"` — `domain_name` → `yamang02.com`, Route53 zone을 `yamang02.com`으로 교체
- `module "frontend"` — aliases를 `admin.yamang02.com` 단독, `certificate_arn` → `f3d39247`

**environments/staging/main.tf**
- `module "frontend"` — aliases에서 `staging.yamangsolution.com` 제거, cert ARN 업데이트

**Terraform import**
- `module.frontend_profile_prod` (또는 별도 module): E22O2QL7DWQJDY import
- `module.frontend_profile_staging`: E2KRLTT5DRFTUA import
- profile 배포용 `aws-frontend` module 인스턴스 추가 또는 별도 tf 파일 작성

**terraform state 정리**
- 구 `yamangsolution.com` Route53 zone이 state에 있다면 `terraform state rm` 처리

## Import 명령어 (production workspace에서 실행)

```bash
# 1. Route53 zone (yamang02.com, 이미 존재)
terraform import module.dns.aws_route53_zone.main Z01437241DGHDFQHEDV82

# 2. ACM 메인 cert (yamang02.com, 18e7ebbd)
terraform import module.dns.aws_acm_certificate.main \
  arn:aws:acm:us-east-1:601326494743:certificate/18e7ebbd-e492-4aa9-9667-9cbec395a66c

# 3. ACM wildcard cert (*.yamang02.com, f3d39247)
terraform import module.dns.aws_acm_certificate.wildcard \
  arn:aws:acm:us-east-1:601326494743:certificate/f3d39247-2fd7-4baa-aa3b-26b0b475363f

# 4. Profile CF distribution (E22O2QL7DWQJDY)
terraform import module.frontend_profile.aws_cloudfront_distribution.main E22O2QL7DWQJDY

# 5. Profile S3 bucket
terraform import module.frontend_profile.aws_s3_bucket.main ai-portfolio-profile-production
```

## Import 명령어 (staging workspace에서 실행)

```bash
# Profile staging CF distribution (E2KRLTT5DRFTUA)
terraform import module.frontend_profile.aws_cloudfront_distribution.main E2KRLTT5DRFTUA

# Profile staging S3 bucket
terraform import module.frontend_profile.aws_s3_bucket.main ai-portfolio-profile-staging
```
