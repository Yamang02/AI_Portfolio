# P03: infra-cleanup-and-terraform-draft

## 목표

- P01에서 발견된 CORS 불일치(staging origin 불일치 등)를 실제 설정에서 해소한다.
- 현재 수동 관리 중인 AWS(S3, CloudFront) 및 GCP(Cloud Run) 리소스에 대한 Terraform 초안을 작성한다.
- Terraform 초안은 `terraform import`로 현재 리소스를 가져올 수 있는 수준으로 작성하되, 실제 인프라를 변경하지 않는다.

## 구현 상세

### 전제 조건

P02 완료 후 진행한다. CORS 수정은 확정된 도메인 전략 기준으로 진행해야 한다.

### 접근 방식

**CORS 정비**:
- P01에서 확인된 staging domain을 기준으로 backend staging `application-staging.yml` CORS allowed-origins 수정
- Production CORS는 P01 검증 결과 정상이면 수정 없음
- Admin staging 네이밍 통일: `admin.staging.yamang02.com` → `staging.admin.yamang02.com`
  - legacy 병행 없이 즉시 전환

**Terraform 초안 작성 범위**:

```
infra/terraform/
  aws/
    s3.tf        — frontend, profile S3 버킷 resource 정의
    cloudfront.tf — frontend, profile CloudFront distribution resource 정의
    variables.tf
    versions.tf
  gcp/
    cloud_run.tf  — backend, staging, chatbot Cloud Run service resource 정의
    variables.tf
    versions.tf
```

- 실제 인프라 변경 없음 (`terraform plan` 검증 수준)
- `terraform import` 명령 주석을 각 resource 위에 기록
- Secret 값은 variable로 추상화, `.tfvars.example` 작성

### 변경 파일

- `backend/src/main/resources/application-staging.yml` — CORS allowed-origins 수정 (P01 결과 기반)
