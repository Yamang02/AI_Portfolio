## 도메인 구조

```
yamang02.com
├── www.yamang02.com         ← 이정준 개인 허브 (운영 중)
├── portfolio.yamang02.com   ← 개인 사이트(포트폴리오 성격) 전환 후보
├── blog.yamang02.com        ← 개인 사이트(블로그 성격) 전환 후보
└── design.yamang02.com      ← YamangDesign 디자인 갤러리 (운영 중)

yamangsolution.com
└── www.yamangsolution.com   ← 현재 포트폴리오 임시 운영 (목표는 비즈니스 전용)
```

---

## 인프라 현황

### 레포 구조와 도메인 매핑

| 디렉토리 | 도메인 | 형태 | 상태 |
|---|---|---|---|
| `profile/` | www.yamang02.com | 순수 HTML 정적 페이지 | 운영 중 |
| `frontend/` | www.yamangsolution.com | React SPA | 운영 중 (과도기), 추후 포트폴리오 도메인으로 이전 |
| `backend/` | ai-portfolio-493721639129.asia-northeast3.run.app | Spring Boot | GCP Cloud Run |

### 서비스별 인프라

```
AWS S3 + CloudFront
├── yamang02.com       ← profile/ 빌드 서빙
└── yamangsolution.com ← frontend/ 빌드 서빙

GCP Cloud Run
├── Spring Boot (백엔드 API)
└── FastAPI (AI 서비스)

Railway
└── PostgreSQL

Cloudflare
├── yamangsolution.com 도메인 구매/DNS 관리
└── Tunnel (로컬 LLM 연결 예정 - 미완)
```

---

## 정비 과제

### 단기
- `yamangsolution.com` 비즈니스 사이트 IA/콘텐츠/구축 착수
- `frontend/` 개인 사이트를 `portfolio.yamang02.com` 또는 `blog.yamang02.com`으로 이전하기 위한 컷오버 계획 수립
- 개인 사이트 스테이징 도메인(`staging.portfolio.yamang02.com` 또는 `staging.blog.yamang02.com`) 운영 규칙 확정
- 과도기 운영 원칙 유지: 비즈니스 사이트 준비 전까지 `www.yamangsolution.com` 유지

### 중기
- Railway PostgreSQL → GCP Cloud SQL 통합
  - 이유: 관리 포인트 축소, GCP 내 네트워크 레이턴시 개선, 고정 비용 절감

### 장기
- Cloudflare Tunnel + 로컬 LLM 연결 완성
  - FastAPI 대체 또는 병행 운영 가능
  - GCP FastAPI 비용 절감 가능

사이트 분리 및 인프라 정비는 DB/시스템 구조 재설계가 선행되어야 함.
브랜드/도메인 구조 확정 → DB 재설계 → 인프라 정비 순서로 진행.
