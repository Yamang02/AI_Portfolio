# P01: infra-audit

## 목표

- AWS CloudFront Distribution의 실제 aliases(도메인 매핑)를 확인하고 기록한다.
- GCP Cloud Run 서비스 목록 및 custom domain 매핑 현황을 확인한다.
- Cloudflare DNS 레코드(yamang02.com, yamangsolution.com)를 확인한다.
- Production/Staging CORS preflight가 실제로 통과하는지 검증한다.
- 감사 결과를 `decisions/D01.infrastructure-domain-source-of-truth.md`에 반영하고, 불일치 항목을 목록화한다.

## 구현 상세

### 접근 방식

`decisions/D01.infrastructure-domain-source-of-truth.md` 섹션 5.1~5.4의 감사 명령을 실행하고, 결과를 원문 문서에 직접 반영한다.

### 확인 항목

**AWS 감사**:
- `aws cloudfront get-distribution --id <ID>` → Aliases 필드로 실제 도메인 확인
  - frontend production CloudFront alias
  - frontend staging CloudFront alias (`staging.yamangsolution.com`인지 `staging.yamang02.com`인지 확정)
  - profile production CloudFront alias
- `aws s3api list-buckets` → S3 버킷명 확인

**GCP 감사**:
- `gcloud run services list --region=asia-northeast3` → 실행 중인 서비스 목록
- `gcloud run domain-mappings list --region=asia-northeast3` → custom domain 매핑 현황
- `gcloud sql instances list` → Cloud SQL 존재 여부 (Railway 전환 상태 확인)

**Cloudflare 감사**:
- Cloudflare Dashboard에서 yamangsolution.com, yamang02.com DNS 레코드 확인
- CNAME/A 레코드가 CloudFront distribution과 일치하는지 확인

**CORS 동작 테스트**:
- Production: `www.yamangsolution.com` → backend API CORS preflight
- Staging: `staging.yamangsolution.com`(또는 `staging.yamang02.com`) → staging backend CORS preflight
