# P02: 아티클 상세 연관 기술카드 표시

## 목표

아티클 상세 API 응답에 연관 기술카드 목록이 포함되고, 아티클 상세 페이지에서 해당 기술카드가 있을 때 별도 섹션으로 표시된다.

## 구현 상세

### 접근

- **백엔드**: 아티클 상세 응답에 `technicalCards` 필드를 추가한다. `ProjectTechnicalCard.articleId`로 역방향 조회하여 해당 아티클에 연결된 기술카드 목록을 포함시킨다.
- **프론트엔드**: `ArticleDetailPage`에 "연관 기술카드" 섹션을 추가한다. 기존 기술카드 렌더링은 `ProjectDetailPage`의 `technicalCards` 섹션 UI를 참조한다.

### 변경 범위

**백엔드**

- `infrastructure/web/controller/ArticleController.java` 또는 관련 응답 DTO: 아티클 상세 응답에 `technicalCards` 필드 추가
- `application/article/GetArticleService.java`: `articleId`로 기술카드 조회 로직 추가
- `domain/article/port/in/GetArticleUseCase.java` 또는 out port: 기술카드 조회 포트 추가 (필요 시)
- `infrastructure/persistence/`: `article_id`로 `ProjectTechnicalCard` 조회 쿼리 추가

**프론트엔드**

- `frontend/src/main/entities/article/model/` (또는 관련 타입 파일): 아티클 타입에 `technicalCards?: ArticleRelatedTechnicalCard[]` 필드 추가
- `frontend/src/main/pages/ArticleDetailPage.tsx`:
  - "연관 기술카드" 섹션 추가 (`id="related-technical-cards"`)
  - TOC에 기술카드 항목 조건부 추가
  - 기술카드 UI는 `ProjectDetailPage`의 카드 렌더링 스타일 참조

## 체크리스트

- [x] 아티클 상세 API 응답에 `technicalCards` 배열이 포함된다 (연결된 카드 없으면 빈 배열)
- [x] `articleId`가 일치하는 기술카드만 반환된다
- [x] 아티클 상세 페이지에서 기술카드가 있을 때 "연관 기술카드" 섹션이 표시된다
- [x] 기술카드가 없는 아티클에서 해당 섹션이 나타나지 않는다
- [x] TOC에 기술카드 항목이 조건부로 포함된다
- [x] TypeScript 타입이 일치하고 컴파일 에러가 없다
