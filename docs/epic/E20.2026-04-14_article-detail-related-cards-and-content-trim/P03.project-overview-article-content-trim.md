# P03: 프로젝트 개요 아티클 1~2문단 압축 (데이터)

## 목표

각 프로젝트의 `projectOverviewArticle.content`가 1~2문단으로 압축되어 Admin에 저장된 상태가 된다. 코드 변경 없이 데이터 작업으로 완료된다.

## 구현 상세

### 접근

Admin UI(`/admin`)의 프로젝트 편집 화면에서 각 프로젝트의 개요 아티클 본문을 직접 수정한다. 핵심 가치 제안과 차별점을 중심으로 1~2문단으로 압축한다.

### 대상 프로젝트

`projectOverviewArticle.content`가 존재하는 모든 프로젝트. Admin 목록에서 순서대로 확인하며 작업한다.

### 압축 기준

- 1문단: 프로젝트가 해결하는 문제 또는 핵심 기능 한 줄 요약
- 2문단(선택): 기술적 접근법 또는 주요 성과 한 줄 요약
- 기술 세부사항, 구현 과정 설명은 기술카드로 이동하거나 삭제

## 체크리스트

- [x] `projectOverviewArticle.content`가 있는 프로젝트 수를 확인한다 (10개, 주요 4개 압축 대상)
- [x] 각 프로젝트의 개요 아티클이 1~2문단으로 저장된다 (야망디자인, Genpresso, AI 포트폴리오 챗봇, SKKU FAC — Docker DB 직접 UPDATE)
- [ ] 작업물 상세 페이지에서 "프로젝트 개요" 섹션이 짧게 렌더링됨을 브라우저에서 확인한다
