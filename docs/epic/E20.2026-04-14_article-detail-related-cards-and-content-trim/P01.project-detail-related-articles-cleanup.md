# P01: 작업물 상세 관련글 섹션 제거 및 코드 정리

## 목표

`ProjectDetailPage`에서 "관련 글" 섹션과 이를 위한 모든 파생 코드가 제거되어, 페이지에 해당 섹션이 더 이상 렌더링되지 않는다.

## 구현 상세

### 접근

`developmentTimelineArticles` 관련 코드를 `ProjectDetailPage.tsx` 내에서 일괄 제거한다. UI 섹션, 상태, TOC 항목, 미사용 import를 순서대로 정리한다.

### 변경 범위

**수정: `frontend/src/main/pages/ProjectDetailPage/ProjectDetailPage.tsx`**

- "관련 글" 섹션(`id="development-timeline"`) 렌더링 코드 제거
- `developmentTimelineArticles` useMemo 제거
- `paginatedArticles` useMemo 제거
- `totalArticlePages` useMemo 제거
- `currentArticlePage` useState 제거
- `setCurrentArticlePage` 리셋 useEffect 제거
- 페이지 변경 시 스크롤 useEffect 제거
- TOC `baseSections`에서 `{ id: 'development-timeline', text: '관련 글', level: 2 }` 항목 제거
- `tocItems` useMemo 의존성에서 `developmentTimelineArticles` 제거
- 미사용 import 제거: `SimpleArticleCard`, `Pagination`, `ARTICLES_PER_PAGE` 상수

## 체크리스트

- [x] 작업물 상세 페이지에 "관련 글" 섹션이 렌더링되지 않는다
- [x] 목차(TOC)에 "관련 글" 항목이 나타나지 않는다
- [x] `developmentTimelineArticles` 관련 상태(useState, useMemo)가 파일에 존재하지 않는다
- [x] `SimpleArticleCard`, `Pagination` import가 제거된다
- [x] TypeScript 컴파일 에러가 없다 (pageConfig.ts 기존 오류와 무관)
- [x] 다른 섹션(개요, 기술카드, 기술스택, 캐러셀, 내비게이션)의 렌더링에 영향이 없다
