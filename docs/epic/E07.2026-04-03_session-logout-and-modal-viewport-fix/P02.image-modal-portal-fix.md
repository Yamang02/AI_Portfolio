# P02: 이미지 모달 React Portal 적용

## 목표

서비스 앱 아티클 본문에서 이미지 클릭 시 모달이 항상 브라우저 뷰포트 정중앙에 표시된다.

## 구현 상세

**접근:** 디자인 시스템 `Modal`이 `ReactDOM.createPortal`(또는 `createPortal` from `react-dom`)을 사용해 `document.body`에 직접 렌더링되도록 변경한다.  
이렇게 하면 `@uiw/react-markdown-preview`나 다른 조상 요소의 CSS stacking context(transform, filter, contain 등)에 영향받지 않으며, `position: fixed`가 항상 뷰포트를 기준으로 동작한다.

**적용 범위:** 이 저장소에서 **어드민**은 `admin/shared/ui` 등 **Ant Design `Modal`**을 사용하므로 P02와 무관하다. 변경의 영향은 **디자인 시스템 `Modal`을 import하는 메인(서비스) 앱 화면** 전부다(아티클 `MarkdownRenderer`, 채팅 안내 모달, 프로젝트 검색 모달 등).

**부가 확인:** Portal로 `body` 직하에 두어도, 다른 전역 레이어가 더 높은 `z-index`를 쓰면 가려질 수 있다. 문제가 있으면 오버레이·콘텐츠 래퍼의 `z-index`를 한 번 확인한다.

**변경 파일:**
- `frontend/src/design-system/components/Modal/Modal.tsx`
  - `react-dom`에서 `createPortal` 사용 (프로젝트 스타일에 맞게 `ReactDOM.createPortal` 호출도 가능)
  - 루트 UI를 `createPortal(..., document.body)`로 감쌈
  - SSR 환경이 없으므로 `document` 가드 불필요

## 체크리스트

- [ ] 아티클 본문 이미지 클릭 시 모달이 뷰포트 정중앙에 표시됨
- [ ] 브라우저 DevTools Elements 패널에서 모달 overlay가 `<body>` 직하에 렌더링됨
- [ ] ESC 키 및 오버레이 클릭으로 모달이 정상적으로 닫힘
- [ ] 디자인 시스템 `Modal`을 쓰는 다른 화면(예: 채팅 안내, 프로젝트 검색)에서 열기·닫기·레이아웃이 이전과 동일하게 유지됨
- [ ] 다른 전역 UI(토스트, 드로어 등) 위에 모달이 올바르게 겹치는지(필요 시 `z-index`만 조정)
- [ ] 모바일 뷰포트(≤767px)에서도 모달이 뷰포트 중앙에 표시됨
