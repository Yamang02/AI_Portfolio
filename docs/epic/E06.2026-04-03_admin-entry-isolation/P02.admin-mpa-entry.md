# P02: Admin 독립 진입점 구성 (MPA)

## 목표

Admin App이 `admin.html`을 독립 진입점으로 가지며, 빌드 산출물에 `admin.html`과 Admin 번들이 포함된다.  
로컬에서는 dev server로 `/admin/*` 경로가 Admin 앱을 렌더링하는 것까지 확인한다.

## 구현 상세

### 접근

Vite Multi-Page App(MPA) 설정을 추가한다. `index.html`(Main)과 `admin.html`(Admin)을 각각 독립 빌드 타겟으로 등록한다. Admin 전용 entry script(`src/admin.tsx`)를 신규 작성하여 `AdminApp`을 마운트한다.

기존 `App.tsx`의 Admin 라우트 제거는 **P03**에서 수행하므로, 이 Phase에서는 **독립 진입점 추가**에 집중한다.

### 운영 URL (참고)

환경별 Admin **공개 URL(FQDN)** 은 팀 내부·배포 설정으로 관리하고, 이 문서에는 적지 않는다. DNS·CloudFront·CORS 연결은 **P04**에서 다룬다. P02에서는 빌드·로컬 검증까지를 완료한다.

### 변경 범위

| 구분 | 파일 | 변경 내용 |
|------|------|----------|
| 신규 | `admin.html` | Admin App HTML 진입점 (`src/admin.tsx` 로드) |
| 신규 | `src/admin.tsx` | `AdminApp` 마운트 entry script |
| 수정 | `vite.config.ts` | `build.rollupOptions.input`에 `admin.html` 추가 |

### vite.config.ts 변경 방향

```ts
build: {
  rollupOptions: {
    input: {
      main: 'index.html',
      admin: 'admin.html',
    },
  },
},
```

### 배포 고려 (P02 체크리스트 수준)

- `npm run build` 후 `dist/admin.html` 및 관련 청크 존재 확인.
- 프로덕션에서 Admin 전용 호스트로의 딥링크는 **P04**의 CloudFront 함수로 `admin.html`을 타게 한다.

## 체크리스트

- [ ] `admin.html`이 존재하며 `src/admin.tsx`를 로드한다.
- [ ] `src/admin.tsx`가 `AdminApp`을 루트 DOM에 마운트한다.
- [ ] `vite.config.ts`에 MPA input이 등록되어 있다.
- [ ] 로컬 dev server에서 `/admin/login` 등 기존 경로로 접근 시 Admin 화면이 렌더링된다.
- [ ] `npm run build` 결과물에 `admin.html`과 Admin 번들이 포함된다.
