# P02: Admin 독립 진입점 구성

## 목표

Admin App이 `admin.html`을 독립 진입점으로 가지며, `/admin/` URL로 직접 접근 시 Main App 라우터를 거치지 않고 렌더링된다.

## 구현 상세

### 접근

Vite Multi-Page App(MPA) 설정을 추가한다. `index.html`(Main)과 `admin.html`(Admin)을 각각 독립 빌드 타겟으로 등록한다. Admin 전용 entry script(`src/admin.tsx`)를 신규 작성하여 `AdminApp`을 마운트한다.

기존 `src/main.tsx`와 `App.tsx`의 Admin 라우트 제거는 P03에서 수행하므로, 이 Phase에서는 Admin 독립 진입점 추가에만 집중한다.

### 변경 범위

| 구분 | 파일 | 변경 내용 |
|------|------|----------|
| 신규 | `admin.html` | Admin App HTML 진입점 (`src/admin.tsx` 로드) |
| 신규 | `src/admin.tsx` | `AdminApp` 마운트 entry script |
| 수정 | `vite.config.ts` | `build.rollupOptions.input`에 `admin.html` 추가 |

### vite.config.ts 변경 방향

```ts
build: {
  rollupOptions: {
    input: {
      main: 'index.html',
      admin: 'admin.html',
    },
  },
},
```

### 배포 고려

Cloud Run(프로덕션)에서 `/admin/` 경로가 `admin.html`로 서빙되도록 nginx 또는 서버 설정 확인이 필요하다. 로컬 Vite dev server는 MPA를 자동 지원한다.

## 체크리스트

- [ ] `admin.html`이 존재하며 `src/admin.tsx`를 로드한다.
- [ ] `src/admin.tsx`가 `AdminApp`을 `#admin-root`(또는 별도 root)에 마운트한다.
- [ ] `vite.config.ts`에 MPA input이 등록되어 있다.
- [ ] 로컬 dev server에서 `/admin/login`에 직접 접근 시 Admin 로그인 화면이 렌더링된다.
- [ ] `npm run build` 결과물에 `admin.html`과 Admin 번들이 포함된다.
