# P04: Admin 전용 호스트·CloudFront·CORS

## 목표

- **Staging**: `https://admin.staging.yamang02.com` 이 동일 프론트 빌드(`admin.html`)를 제공하고, 딥링크·새로고침이 깨지지 않는다.
- **Production**: `https://admin.yamang02.com` 동일.
- 스테이징·프로덕션 백엔드 **CORS**에 위 Origin이 포함된다.
- 필요한 변경이 **Terraform** 및 DNS에 반영되고, `terraform plan`으로 의도한 수준만 변경된다.

## 배경

S3 + CloudFront 구성에서는 단일 Distribution에 **SPA 폴백**(403 등 → `/index.html`)이 있으면, Admin 전용 호스트에서도 동일 폴백이 적용될 수 있다.  
Admin 전용 **호스트 이름**으로 구분하여, Viewer-request 단계에서 해당 Host일 때 HTML 내비게이션 요청을 **`/admin.html`** 로 맞춘다 (공유 `assets/` 등 정적 경로는 제외).

## 구현 상세

### 1. DNS·인증서

- `admin.staging.yamang02.com`, `admin.yamang02.com` 이 **ACM( us-east-1, CloudFront용)** 및 **Route53** 검증 규칙과 맞는지 확인한다.  
  와일드카드 인증서가 이미 `*.yamang02.com` 형태로 두 이름을 커버하는지 검토한다. 커버하지 않으면 SAN 추가 또는 별도 인증서 발급 후 Distribution에 연결한다.

### 2. CloudFront

- 기존 Staging / Production 프론트 Distribution에 **Alternate domain names (CNAME)** 로 Admin 호스트를 추가한다 (별도 Distribution을 새로 만들지 않는 것을 권장 — 동일 S3 오리진·동일 빌드).
- **Viewer-request CloudFront Function** (또는 동일 효과의 Edge 로직):  
  - `Host` 가 `admin.staging.yamang02.com` 또는 `admin.yamang02.com` 일 때  
  - 요청 경로가 Admin 정적 자산이 아니면 오리진 요청 URI를 **`/admin.html`** 로 정규화한다 (쿼리 문자열 유지).  
  - 자산 판별 규칙은 빌드 산출물 구조(`assets/` 등)에 맞게 정한다.

Terraform: `aws_cloudfront_function`, `aws_cloudfront_distribution` 의 `function_association` 등 기존 모듈(`modules/aws-frontend`)에 옵션으로 추가 가능하면 모듈화한다.

### 3. Route53

- `admin.staging.yamang02.com` → Staging CloudFront Distribution (별칭 A/AAAA).  
- `admin.yamang02.com` → Production CloudFront Distribution.  
  (실제 레코드는 팀의 Hosted Zone 설정을 따른다.)

### 4. 백엔드 CORS

- `application-staging.yml` (및 프로덕션 설정)의 `spring.web.cors.allowed-origins` (또는 동등 설정)에 다음을 추가한다.  
  - `https://admin.staging.yamang02.com`  
  - `https://admin.yamang02.com`  
- 환경 변수로 오리진을 주입하는 패턴이 있으면 동일하게 확장한다.

### 5. CI/CD

- 프론트 워크플로는 기존처럼 `dist/` 전체를 S3에 동기화하면 된다.  
- 배포 후 Admin 호스트로 **무효화**가 필요하면 CloudFront invalidation 경로에 `admin.html`, `/assets/*` 등을 포함하는지 검토한다 (기존 `/*` 무효화면 충분할 수 있음).

### 6. 검증

- Staging: `https://admin.staging.yamang02.com/admin/login` 직접 접속·새로고침 시 Admin 로그인 화면 유지.  
- Production: `https://admin.yamang02.com` 동일.  
- 메인 사이트 URL에서 Admin 번들이 로드되지 않음(P03과 합쳐 확인).

## 체크리스트

- [ ] ACM·CloudFront·Route53 상 Admin 호스트가 Distribution에 연결되어 있다.
- [ ] Edge 함수로 Admin 호스트에서 HTML 요청이 `admin.html`을 타며, 딥링크가 동작한다.
- [ ] 백엔드 CORS에 두 Admin Origin이 등록되어 API 호출이 실패하지 않는다.
- [ ] Terraform 적용 후 `terraform plan`이 예상 범위만 보인다 (환경별로 실행).
- [ ] Staging·Production 각각에서 위 검증 URL을 수동 확인했다.
