# P03: admin-technical-card-editor-ui

## 목표

- Admin 프로젝트 편집 화면에서 Technical Card를 직접 관리할 수 있게 한다.
- 카드 생성/수정/삭제/정렬/핀 제어를 단일 편집 흐름으로 제공한다.

## 구현 상세

### 전제 조건

- P02 백엔드 계약 및 API 구현이 완료되어야 한다.
- P01 FSD 정리가 완료되어야 한다.
- Admin의 기존 FSD 경계(`pages -> features -> entities -> shared`)를 유지한다.

### 접근 방식

1. 편집 UI 구성
- `ProjectEdit` 하위에 Technical Card 편집 섹션 추가
- 카드 리스트/폼/정렬 컨트롤/핀 토글을 분리된 feature 컴포넌트로 구성

2. 상태/저장 흐름
- 기존 프로젝트 저장과 카드 저장을 일관된 payload로 통합
- 서버 검증 실패/부분 실패 메시지 처리 정책 고정

3. 사용성
- 카드 미리보기 또는 최소한의 구조화 뷰 제공
- 정렬 변경 시 즉시 시각 피드백 제공

- [ ] 기존 프로젝트 편집 기능 회귀 확인
