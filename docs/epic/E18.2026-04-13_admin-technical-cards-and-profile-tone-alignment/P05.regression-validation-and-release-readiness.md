# P05: regression-validation-and-release-readiness

## 목표

- Admin 편집과 Main 렌더링 전환 이후 회귀를 검증한다.
- 릴리즈 전 데이터/계약/UI 품질 게이트를 통과한다.

## 구현 상세

### 검증 범위

1. Admin 기능 회귀
- 프로젝트 생성/수정/삭제
- Technical Card CRUD/정렬/핀 저장

2. Main 기능 회귀
- 프로젝트 상세 로딩/오류/빈 상태
- `projectOverviewArticle`/Technical Card/관련글/기술스택 표시

3. 품질 게이트
- Frontend 테스트(`npm run test:coverage`)와 lint/build 확인
- 필요한 Sonar 재분석 및 남은 이슈 triage

### 릴리즈 체크

- 스키마/API/UI 계약이 문서와 일치하는지 확인
- `readme` 제거 전 fallback이 남아 있지 않은지 확인

## 체크리스트

- [ ] Admin E2E 핵심 시나리오 통과
- [ ] Main 상세 회귀 시나리오 통과
- [ ] 테스트/린트/빌드 통과
- [ ] Sonar 이슈 상태 업데이트
- [ ] 릴리즈 노트/에픽 상태 갱신
