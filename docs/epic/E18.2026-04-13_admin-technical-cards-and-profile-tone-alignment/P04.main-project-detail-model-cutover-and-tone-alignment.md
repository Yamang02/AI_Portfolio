# P04: main-project-detail-model-cutover-and-tone-alignment

## 목표

- 메인 프로젝트 상세 렌더링을 E17 콘텐츠 모델로 전환한다.
- 포트폴리오 페이지의 카피/비주얼 톤을 `profile/` 프로젝트와 정렬한다.
- "그리운 경찰 감성체"를 제거하고 일관된 문체/톤 가이드를 적용한다.

## 구현 상세

### 전제 조건

- P02 백엔드 계약과 P03 Admin 입력 흐름이 확정되어야 한다.
- 디자인 토큰/공통 컴포넌트 사용 원칙(E15)을 유지한다.

### 접근 방식

1. 데이터 모델 전환
- `ProjectDetailPage`에서 `project.readme` 의존 제거
- `projectOverviewArticle` 인라인 렌더링 + `technicalCards` 섹션 렌더링으로 전환

2. TOC/앵커/내비게이션 정렬
- 개요/기술카드/관련글/기술스택 순서를 새 모델 기준으로 재정의
- 카드 앵커와 TOC 연결 규칙 확정

3. 톤 정렬
- `profile/`의 카피 톤과 충돌하는 표현 제거
- 타이포/간격/컬러 톤을 profile 방향으로 맞추되 main 런타임 특성은 유지
