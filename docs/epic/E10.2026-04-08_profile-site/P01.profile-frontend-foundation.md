# P01: 프로필 프론트엔드 기반 구성

## 목표

- `profile/` 디렉토리가 모노레포 내에 독립 Vite + React + TypeScript 프로젝트로 존재한다.
- `content.json` 스키마가 확정되고 초기 데이터가 작성된다.
- `npm run dev` 로 개발 서버가 실행되고 `content.json` 데이터가 렌더링된다.
- `npm run build` 로 빌드 산출물이 `profile/dist/` 에 생성된다.

## 구현 상세

### 디렉토리 구조

```
profile/
  public/
    content.json       ← identity + works 콘텐츠
  src/
    main.tsx
    App.tsx
    api/
      content.ts       ← content.json fetch 레이어 (향후 API 교체 지점)
    types/
      profile.ts       ← ProfileIdentity, ProfileWork 타입 정의
    components/
    styles/
  package.json
  vite.config.ts
  tsconfig.json
```

### `content.json` 스키마

```json
{
  "identities": [
    {
      "key": "teacher",
      "keyword": "문제의식",
      "narrative": "..."
    }
  ],
  "works": [
    {
      "id": "pw-001",
      "title": "바이브코딩 강의",
      "description": "...",
      "externalUrl": "...",
      "isVisible": true,
      "sortOrder": 1
    }
  ]
}
```

### fetch 레이어 추상화

`api/content.ts` 가 `content.json` 을 fetch하는 단일 진입점. 향후 백엔드 API로 교체 시 이 파일만 수정한다.

### 생성 파일

- `profile/package.json`
- `profile/vite.config.ts`
- `profile/tsconfig.json`
- `profile/index.html`
- `profile/src/main.tsx`
- `profile/src/App.tsx`
- `profile/src/types/profile.ts`
- `profile/src/api/content.ts`
- `profile/public/content.json`

## 체크리스트

- [ ] `npm install` 이 오류 없이 완료된다
- [ ] `npm run dev` 로 개발 서버가 실행된다
- [ ] `content.json` 데이터가 정상적으로 로드된다
- [ ] `npm run build` 로 `dist/` 산출물이 생성된다
- [ ] `frontend/` 빌드에 영향을 주지 않는다
