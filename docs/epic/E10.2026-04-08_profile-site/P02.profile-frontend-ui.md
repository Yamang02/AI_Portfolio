# P02: 프로필 프론트엔드 UI 구현

## 목표

- 확정된 디자인 기준으로 세 정체성 섹션, 교차 결과물 섹션, CTA가 구현된다.
- 모바일·데스크톱 반응형이 동작한다.

## 전제 조건

- 디자인 에픽(E11 예정)에서 시각 디자인이 확정된 이후 진행한다.

## 구현 상세

### 페이지 구조

```
[상단] 한 줄 소개

[중단] 세 정체성 병렬 배치
  ┌─────────────┬──────────────┬─────────────┐
  │ 문제의식     │ 구현가능성    │ 영감         │
  │ narrative   │ narrative    │ narrative   │
  └─────────────┴──────────────┴─────────────┘

[하단] 교차 결과물
  isVisible=true, sortOrder 순 렌더링

[CTA] → 포트폴리오 사이트 (하드코딩 상수)
```

### 생성 파일

- `profile/src/components/IdentitySection.tsx`
- `profile/src/components/IdentityCard.tsx`
- `profile/src/components/WorkSection.tsx`
- `profile/src/components/WorkCard.tsx`
- `profile/src/components/CtaSection.tsx`
- `profile/src/styles/` — 페이지 전용 스타일

## 체크리스트

- [ ] 세 정체성 카드가 `content.json` 데이터로 렌더링된다
- [ ] 교차 결과물이 `sortOrder` 순으로 렌더링된다
- [ ] `isVisible: false` 항목은 노출되지 않는다
- [ ] 각 결과물의 `externalUrl` 링크가 동작한다
- [ ] CTA가 포트폴리오 사이트로 연결된다
- [ ] 모바일(375px)·데스크톱(1280px)에서 레이아웃이 깨지지 않는다
