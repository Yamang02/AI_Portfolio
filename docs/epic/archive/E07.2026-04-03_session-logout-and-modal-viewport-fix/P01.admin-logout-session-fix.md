# P01: 어드민 로그아웃 세션/쿠키 처리 수정

## 목표

어드민 로그아웃 시 서버 세션과 브라우저 SESSION 쿠키가 동시에 제거되고, 프론트엔드가 로그인 페이지로 확실하게 이동한다.

## 구현 상세

### 백엔드

**접근:** 로그아웃 엔드포인트에서 `HttpServletResponse`를 파라미터로 받아, `Set-Cookie: SESSION=; Max-Age=0` 헤더를 응답에 추가한다.  
쿠키 속성은 `SessionConfig`에 설정된 값(path `/`, SameSite, Secure, Domain)과 일치해야 크로스도메인 환경에서 브라우저가 쿠키를 삭제한다.

**구현 선택:** 응답에 `Set-Cookie`를 직접 써도 되고, 프로젝트에 맞다면 Spring Session이 사용하는 `CookieSerializer`로 만료 쿠키를 추가하는 방식도 가능하다. 어떤 방식이든 **세션 쿠키와 동일한 이름·도메인·경로·플래그**가 나가야 한다.

**변경 파일:**
- `backend/src/main/java/com/aiportfolio/backend/infrastructure/web/admin/session/AdminSessionManager.java`
  - `clearSession(HttpSession, HttpServletResponse)` 오버로드 또는 기존 메서드 시그니처 변경: 세션 무효화 후 response에 만료 쿠키 헤더 작성
- `backend/src/main/java/com/aiportfolio/backend/infrastructure/web/admin/controller/AdminAuthController.java`
  - `logout()` 파라미터에 `HttpServletResponse` 추가, `AdminSessionManager.clearSession(session, response)` 호출

### 프론트엔드

**접근:** `logout()` 함수 내에서 `invalidateQueries` 대신 `queryClient.removeQueries`를 사용해 캐시를 즉시 제거하고, 세션 재요청이 트리거되지 않도록 한다. 이후 명시적 페이지 이동(`window.location.href`)으로 SPA 상태를 완전히 초기화한다.

**하드 네비게이션 URL:** 어드민 앱은 E06 이후 전용 도메인에서 동작하므로, **현재 페이지와 동일 origin**이면 상대 경로 `/admin/login`이면 된다. 빌드·배포 방식으로 origin이 달라질 여지가 있으면, 해당 환경의 어드민 베이스 URL을 쓰는 편이 안전하다.

**변경 파일:**
- `frontend/src/admin/hooks/useAuth.ts`
  - `logout()`: `invalidateQueries` → `removeQueries(['admin-session'])`로 교체
  - 쿼리 제거 후 `window.location.href = '/admin/login'` 등 로그인 URL로 하드 네비게이션
- `frontend/src/admin/shared/components/AdminLayout.tsx`
  - `handleLogout()`의 `navigate('/admin/login')` 제거 (useAuth 내부에서 처리)

## 체크리스트

- [ ] 로그아웃 API 응답 헤더에 `Set-Cookie: SESSION=; Max-Age=0` 포함됨
- [ ] 로그아웃 후 브라우저 DevTools에서 SESSION 쿠키가 삭제됨
- [ ] 로그아웃 클릭 시 로그인 페이지로 이동함
- [ ] 로그인 페이지 이동 후 뒤로가기로 어드민 대시보드 접근 시 로그인 페이지로 리다이렉트됨
- [ ] 로그아웃 후 `/api/admin/auth/session` 재요청이 발생하지 않음 (Network 탭 확인)
