# P03: Generic Exception 정리 (S112)

## 목표

- `throw new RuntimeException` 등 generic exception 사용 47건을 적절한 커스텀/라이브러리 예외로 교체한다
- 기존 예외 처리 흐름(GlobalExceptionHandler 등)과 정합성을 유지한다

## 대상 이슈

| 규칙 | 건수 | 설명 |
|------|------|------|
| S112 | 47 | Generic exception(RuntimeException, Exception 등) 대신 구체적 예외 사용 |

## 구현 상세

### 접근 방식

1. 기존 프로젝트의 커스텀 예외 체계 파악 (어떤 예외 클래스가 이미 존재하는지)
2. 47건을 발생 컨텍스트별로 분류:
   - 이미 적절한 커스텀 예외가 있는 경우 → 교체
   - 새 커스텀 예외가 필요한 경우 → 도메인/레이어에 맞게 생성
   - 표준 라이브러리 예외로 충분한 경우 → `IllegalArgumentException`, `IllegalStateException` 등 사용
3. Hexagonal Architecture 레이어 경계 준수:
   - Domain 예외는 domain 패키지에
   - Application 예외는 application 패키지에
   - Infrastructure 예외는 infrastructure 패키지에서 domain 예외로 변환

### 변경 범위

- **Modify**: S112 해당 47개 파일/위치
- **Create**: 필요한 커스텀 예외 클래스 (분석 후 확정)
