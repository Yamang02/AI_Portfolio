# P01: 공유 상수 분리

## 목표

Main App 소스가 `@/admin/entities/article`을 import하지 않는다.
`ARTICLE_CATEGORIES`가 `src/shared/`에 위치하며 Main과 Admin 양쪽에서 참조된다.

## 구현 상세

### 접근

`ARTICLE_CATEGORIES`는 아티클 카테고리 목록 상수로, Admin 전용 개념이 아니다.
`src/shared/article/` 하위로 이동하고, 기존 참조 경로를 일괄 교체한다.
Admin 쪽은 `shared`를 직접 import하며, Admin 엔티티에서의 re-export는 하지 않는다 (에픽 Readme 결정과 동일).

### 변경 범위

| 구분 | 파일 | 변경 내용 |
|------|------|----------|
| 신규 | `src/shared/article/index.ts` | `ARTICLE_CATEGORIES` 상수 정의 이전 |
| 수정 | `src/admin/entities/article/index.ts` | `ARTICLE_CATEGORIES` 정의 제거; Admin에서 필요 시 `@/shared/article` 직접 import (엔티티에서 재노출하지 않음) |
| 수정 | `src/main/pages/ArticleDetailPage.tsx` | import 경로 교체 |
| 수정 | `src/main/features/article-view/ui/ArticleFilterBar.tsx` | import 경로 교체 |
| 수정 | `src/main/features/article-view/ui/ArticleTable.tsx` | import 경로 교체 |
| 수정 | `src/main/features/article-view/ui/CategoryFilterBar.tsx` | import 경로 교체 |

## 체크리스트

- [ ] `src/shared/article/index.ts`에 `ARTICLE_CATEGORIES`가 정의되어 있다.
- [ ] Main 페이지 4곳의 import가 `@/shared/article`을 참조한다.
- [ ] `@/admin/entities/article` import가 Main 소스에 남아 있지 않다.
- [ ] Admin에서 `ARTICLE_CATEGORIES`를 쓰는 코드는 `@/shared/article`을 참조하며 빌드가 통과한다.
- [ ] `npm run build` 성공.
