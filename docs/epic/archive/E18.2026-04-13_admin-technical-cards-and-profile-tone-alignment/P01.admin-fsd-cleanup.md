# P01: admin-fsd-cleanup

## 목표

- Admin 프론트엔드의 FSD 경계 위반과 데드코드를 제거한다.
- P03 UI 작업 전에 `shared/ui` 중복 파일과 `components/` 디렉토리 오염을 정리한다.

## 배경

코드베이스 분석에서 발견된 구조 문제:

1. `components/projects/ProjectList.tsx` — FSD 위반. 라우팅 진입점인데 `components/`에 있음.
   `pages/ProjectList.tsx`는 이를 감싸는 wrapper인데 `AdminApp.tsx`에서 라우팅하지 않아 데드코드.

2. `shared/ui/StatsCards.tsx` — `StatsCards/index.tsx`와 중복. Node 모듈 해석 시 디렉토리가 우선되어 플랫 파일은 절대 로드되지 않음. 두 파일의 인터페이스(`StatItem` vs `StatCardData`)가 달라 혼선 유발.

3. `shared/ui/SearchFilter.tsx` — 동일 패턴. `SearchFilter/index.tsx`에 가려진 데드코드.

## 구현 상세

### 전제 조건

- 변경 후 기존 Admin 기능(프로젝트 목록/편집, TechStack, Career 등) 정상 동작 확인 필요.

### 접근 방식

1. ProjectList FSD 정리
   - `components/projects/ProjectList.tsx` 내용을 `pages/ProjectList.tsx`로 이전
   - `AdminApp.tsx` import를 `pages/ProjectList`로 변경
   - `admin/components/` 디렉토리 삭제

2. StatsCards 중복 제거
   - `StatsCards.tsx`(플랫)와 `StatsCards/index.tsx`(디렉토리) 내용 비교 후 디렉토리 버전으로 통합
   - 플랫 파일 삭제, import 경로 영향 없음 (barrel 경유)

3. SearchFilter 중복 제거
   - 동일 처리
