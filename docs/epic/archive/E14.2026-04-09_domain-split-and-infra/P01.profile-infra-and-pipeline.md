# P01: profile 인프라 및 배포 파이프라인

## 목표

- profile용 S3 bucket(production, staging) 2개가 생성된다
- profile용 CloudFront distribution(production, staging) 2개가 생성되고 각각 `www.yamang02.com`, `staging.yamang02.com`에 연결된다
- 기존 production/staging CloudFront에서 해당 도메인 alternate domain이 제거된다
- Route 53 레코드가 새 CloudFront를 가리킨다
- GitHub Actions 워크플로우가 profile production/staging을 빌드·배포하며 EmailJS 키가 주입된다

## 구현 상세

### 현재 인프라 (변경 전)

| 리소스 | ID / 이름 | 현재 alternate domains |
|--------|-----------|----------------------|
| CloudFront production | `E384L5ALEPZ14U` | `yamang02.com`, `www.yamang02.com`, `admin.yamang02.com` |
| CloudFront staging | `E7KKBCETIHDH6` | `staging.yamang02.com`, `admin.staging.yamang02.com` |
| ACM `*.yamang02.com` | `f3d39247-...` | ISSUED — profile에 재활용 가능 |

### 접근 (AWS CLI 순서)

1. **S3 bucket 2개 생성**
   - `ai-portfolio-profile-production`
   - `ai-portfolio-profile-staging`
   - 퍼블릭 액세스 차단 + CloudFront OAC 방식

2. **CloudFront OAC 생성** (production/staging 공용 또는 각각)

3. **CloudFront distribution 2개 생성**
   - production: alternate domain `www.yamang02.com`, `yamang02.com`
     - cert: `*.yamang02.com` (`f3d39247-...`) + `yamang02.com` 인증서 병행 사용
   - staging: alternate domain `staging.yamang02.com`
     - cert: `*.yamang02.com` (`f3d39247-...`)
   - default root object: `index.html`
   - 403/404 custom error → `/index.html` (정적 SPA 대응)

4. **S3 bucket policy 업데이트** (각 CloudFront OAC 허용)

5. **기존 CloudFront alternate domain 제거** (순서 중요 — 먼저 제거 후 신규 추가)
   - `E384L5ALEPZ14U`: `yamang02.com`, `www.yamang02.com` 제거 (`admin.yamang02.com` 유지)
   - `E7KKBCETIHDH6`: `staging.yamang02.com` 제거 (`admin.staging.yamang02.com` 유지)

6. **Route 53 레코드 업데이트**
   - `yamang02.com` A/AAAA → 새 profile production CloudFront
   - `www.yamang02.com` A/AAAA → 새 profile production CloudFront
   - `staging.yamang02.com` A → 새 profile staging CloudFront

7. **GitHub repository variables 추가**
   - `PROFILE_AWS_S3_NAME` = `ai-portfolio-profile-production`
   - `PROFILE_AWS_CLOUDFRONT_DISTRIBUTION_ID` = (신규 production ID)
   - `PROFILE_STAGING_AWS_S3_NAME` = `ai-portfolio-profile-staging`
   - `PROFILE_STAGING_AWS_CLOUDFRONT_DISTRIBUTION_ID` = (신규 staging ID)
   - `VITE_EMAILJS_SERVICE_ID`, `VITE_EMAILJS_TEMPLATE_ID`, `VITE_EMAILJS_PUBLIC_KEY` 확인

8. **GitHub Actions 워크플로우 정비**
   - `profile-production-aws.yml` (이미 생성, 변수명 확인)
   - `profile-staging-aws.yml` 신규 생성

### 변경 범위

| 파일 | 작업 |
|------|------|
| `.github/workflows/profile-production-aws.yml` | 변수명 확인/수정 |
| `.github/workflows/profile-staging-aws.yml` | 신규 생성 |
| `profile/src/emailjs-config.ts` | 완료 |
| `profile/index.html` | EmailJS CDN + module script 완료 |
| AWS (CLI) | S3 ×2, OAC, CloudFront ×2, S3 policy ×2, 기존 CF 수정, Route 53 |
| GitHub Settings | repository variables |
