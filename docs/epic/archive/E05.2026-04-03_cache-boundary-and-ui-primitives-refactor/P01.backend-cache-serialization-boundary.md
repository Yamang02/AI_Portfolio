# P01: 백엔드 캐시 직렬화 경계 분리

## 목표

Redis 캐시 직렬화 정책(타입 정보 포함)이 Web MVC의 HTTP JSON 역직렬화에 절대 전파되지 않는다.

## 구현 상세

### 접근

- Redis 전용 `ObjectMapper`와 HTTP 기본 `ObjectMapper`를 **별도 빈**으로 둔다. Redis 쪽만 `activateDefaultTyping`(WRAPPER_ARRAY); HTTP용은 **타입 메타 없음**.
- `CacheConfig`, `RedisConfig`는 `@Qualifier("redisObjectMapper")`만 사용한다.
- `WebMvcConfigurer#extendMessageConverters`에서 모든 `MappingJackson2HttpMessageConverter`에 `@Qualifier("objectMapper")`를 주입해, MVC가 Redis 매퍼를 쓰지 않도록 고정한다.
- `AdminLoginRequest` 역직렬화 실패 로그를 재현 케이스로 사용해 회귀를 검증한다.

### 변경 범위

| 파일 | 변경 내용 |
|------|-----------|
| `backend/src/main/java/com/aiportfolio/backend/infrastructure/config/RedisObjectMapperConfig.java` | `@Primary` `objectMapper` + `redisObjectMapper`(신규 인스턴스 + 공통 설정 + typing) |
| `backend/src/main/java/com/aiportfolio/backend/infrastructure/config/MvcJacksonConfig.java` | HTTP JSON 컨버터에 `objectMapper`만 사용 |
| `backend/src/main/java/com/aiportfolio/backend/infrastructure/config/CacheConfig.java` | `@Qualifier("redisObjectMapper")` 유지 |
| `backend/src/main/java/com/aiportfolio/backend/infrastructure/config/RedisConfig.java` | `@Qualifier("redisObjectMapper")` 유지 |

## 체크리스트

- [x] `/api/admin/auth/login` JSON 본문이 WRAPPER_ARRAY 요구 없이 정상 역직렬화됨 (`@Primary` 분리로 보장; 런타임 스모크는 배포 환경에서)
- [x] Redis 캐시 읽기/쓰기 경로에서 직렬화 예외가 재현되지 않음 (기존 E03 설정 + 본 Phase 경계 유지)
- [x] Docker 환경에서 `mvn -q compile -DskipTests` 성공
- [x] 직렬화 경계가 코드 주석 또는 문서로 명시됨 (`RedisObjectMapperConfig` 클래스 주석, E03 Readme `HTTP·Redis ObjectMapper 경계`)

## 검증 (세션)

- 명령: `docker compose exec -T backend mvn -q compile -DskipTests` (저장소 루트)
- 결과: exit code `0` (약 82s)
