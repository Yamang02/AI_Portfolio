# 컬러 시스템

업계 표준 컬러 시스템 구조를 따르는 디자인 토큰입니다.

## 구조 개요

컬러 시스템은 두 가지 레벨로 구성됩니다:

1. **Primitive Tokens (기본 토큰)**: 기본 컬러 팔레트
   - `brandScale`: 브랜드 그린 기반 색상 스케일
   - `grayScale`: 중립 회색 스케일
   - `semanticColors`: 의미 기반 색상 스케일 (blue, green, amber, red, purple, yellow)

2. **Semantic Tokens (의미 토큰)**: 의미 기반 컬러
   - `brandSemantic`: 브랜드 컬러 의미 토큰
   - `lightModeSemantic`: 라이트 모드 의미 토큰
   - `darkModeSemantic`: 다크 모드 의미 토큰
   - `specialSemantic`: 특수 목적 의미 토큰 (히어로, Featured 등)

## 단일 소스 원칙

- **CSS 변수** (`globals.css`의 `--color-*`)가 모든 컬러 값의 단일 소스입니다
- Semantic tokens는 CSS 변수를 참조합니다 (`var(--color-*)`)
- TypeScript 코드에서는 semantic tokens를 사용하는 것을 권장합니다

## 사용 가이드

### Primitive Tokens 사용

기본 컬러 팔레트는 직접 사용하지 않고, semantic tokens를 통해 사용합니다.

```typescript
// ❌ 권장하지 않음
import { brandScale } from '@design-system/tokens/colors';
const color = brandScale[400];

// ✅ 권장
import { brandSemantic } from '@design-system/tokens/colors';
const color = brandSemantic.primary; // var(--color-primary)
```

### Semantic Tokens 사용

의미 기반 토큰을 사용하면 테마 변경에 자동으로 대응됩니다.

```typescript
import { lightModeSemantic, brandSemantic } from '@design-system/tokens/colors';

// 배경색
const bgColor = lightModeSemantic.background.primary; // var(--color-bg-primary)

// 텍스트 색상
const textColor = lightModeSemantic.text.primary; // var(--color-text-primary)

// 브랜드 컬러
const primaryColor = brandSemantic.primary; // var(--color-primary)
```

### CSS에서 직접 사용

CSS 모듈이나 인라인 스타일에서 CSS 변수를 직접 사용할 수 있습니다.

```css
.my-component {
  background-color: var(--color-bg-primary);
  color: var(--color-text-primary);
  border: 1px solid var(--color-border-default);
}
```

## 마이그레이션 가이드

기존 코드에서 새로운 컬러 시스템으로 마이그레이션하는 방법:

### 1. Legacy exports 사용 중단

```typescript
// ❌ 구버전
import { brandColors, lightModeColors } from '@design-system/tokens/colors';

// ✅ 신버전
import { brandSemantic, lightModeSemantic } from '@design-system/tokens/colors';
```

### 2. 하드코딩된 색상 값 제거

```typescript
// ❌ 하드코딩
const color = '#7FAF8A';

// ✅ Semantic token 사용
const color = brandSemantic.primary;
```

### 3. CSS 변수 직접 사용

```css
/* ✅ 권장 */
.my-component {
  color: var(--color-text-primary);
}
```

## 접근성 고려사항

- 모든 텍스트는 WCAG AA 기준을 충족합니다 (대비율 4.5:1 이상)
- 상태 색상(성공, 경고, 오류)은 색상뿐만 아니라 아이콘이나 텍스트로도 표시됩니다
- 다크 모드에서도 동일한 접근성 기준을 유지합니다
